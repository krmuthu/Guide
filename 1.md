**Embedding models:**

**Embedding routing is SOURCE-BASED, not per-file-extension:**

**nomic-embed-code mandatory prefixes:**

- Query: `Represent this query for searching relevant code: <query>`

**Bootstrap pipeline (`src/ingestion/bootstrap.ts`, entry: `npm run bootstrap`):**

- **Phase 1 — Initialization:** DB pool, tree-sitter WASM parser pool (6 languages), Bitbucket rate limiter, feature flags (`USE_KNOWLEDGE_GRAPH`, `USE_CONTEXTUAL_EMBEDDINGS`, `DEFER_RICH_HEADERS`), llama-server health checks.
- **Phase 2 — Repo Loop** (parallel, `REPO_CONCURRENCY`): for each of 47 repos:
  - 2a. Clone/Pull (Bitbucket token auth)
  - 2b. Metadata via ProjectAnalyzer (tech_stack, project_type)
  - 2c. Dependencies (package.json, pom.xml, build.gradle, requirements.txt, go.mod)
  - 2d. Sync strategy (full / incremental via `git diff --name-status` / skip)
  - 2e. File ingestion — CAST tree-sitter chunking, parallel via `pLimit(INGEST_CONCURRENCY)`, SHA-256 dedup, WriteQueue micro-batches (100–2000/tx, flush every 500ms)
  - 2f. Stale cleanup
  - 2g. Knowledge graph — AST nodes/edges, Katz centrality, co-change from git history, Louvain communities. Full rebuild if >50 files or >15% changed; otherwise incremental.
  - 2h. Contextual headers (optional, `DEFER_RICH_HEADERS=true`) — Qwen generates 2-3 sentence headers using `getEnrichmentContext`. Post-graph because callers/callees aren’t available before.
  - 2i. Test coverage mapping (naming patterns)
  - 2j. Env var scanning (process.env, os.environ, System.getenv)
  - 2k. Metrics → `ingestion_metrics`
- **Phase 3 — Post-Bootstrap:** conditional VACUUM ANALYZE, HNSW warm-up, failure report.

**Knowledge graph (IGraphClient):**
Dual backend — Apache AGE (`USE_AGE_GRAPH=true`) or relational tables fallback. IGraphClient abstraction makes both transparent.
Node types: Function, Class, File, Module, PR, Developer.
Edge types: calls, imports, extends, implements, uses_type, co_change, community, modifies, authored_by, tests.
Methods: insertNodes, queryNodes, findSymbol, insertEdges, queryEdges, getCallChain, getReverseCallChain, computeCentrality, getEnrichmentContext, getStats.
GraphNode stores: decorators, fields, signature, start_line, end_line, centrality_score (pre-computed Katz).

**Database tables:** projects · chunks · vec_chunks · fts_chunks · files · file_imports · graph_nodes / AGE vertices · graph_edges / AGE edges · test_coverage · env_vars · ingestion_metrics · health_score_reports · health_score_jobs · bootstrap_progress.

**Query pipeline (`src/lib/hybrid_search.ts`, 16 stages):**

1. Cache check (5 min TTL)
1. Embed query (nomic prefix for code, arctic prompt_name for prose)
1. Composite vector expansion (avg with top-5 neighbors, alpha=0.7)
1. Adaptive ef_search (20 identifiers / 60 default / 100 complex)
1. PARALLEL: Dense ANN (top-80) + BM25 FTS (top-80)
1. Source-balanced minimum (`MIN_CODE_IN_CANDIDATES=5`)
1. Dominant-winner short-circuit (≥0.95 → skip reranking)
1. RRF fusion (k=60)
1. Graph expansion via IGraphClient (calls, imports, uses_type, implements, co_change)
1. Katz centrality prominence (0.7 semantic + 0.3 centrality)
1. Score-ratio gating (<75% of top)
1. Pre-rerank dedup
1. Bi-encoder rerank: CodeRankEmbed 80→20 (~400ms)
1. Cross-encoder rerank: bge-reranker-v2-m3 20→5 (~1.5s)
1. MMR diversification (lambda=0.7)
1. Return original content (NOT headers) to MCP/Windsurf

**camelCase BM25 problem & fixes:**
BM25 returns 0 for camelCase tokens (`useState`, `getOrderById`). Code chunks then get vector-only RRF scores and are squeezed out by dual-scored prose.

- Fix 1 (implemented): source-balanced minimum injects code chunks when underrepresented.
- Fix 2: `code_tokenize()` SQL splits camelCase before FTS → `content_tsv_code` column.
- Fix 3: `expandCamelCase()` at query time, OR (`|`) semantics for code BM25.

**MCP tools for Windsurf AI:** `get_code_context`, `analyze_blast_radius`, `find_symbol`, `suggest_related_files`, `get_ticket_context` (plus the broader 55-tool suite — search, navigation, knowledge graph, Jira, Bitbucket, scaffolding, PR review, pipeline, security, agent retrieval, SCIP, API dependency).

**Architecture Health Score (midnight job, REST API + HTML dashboard, NOT MCP tools):**
Three layers — AST static analyzers, IGraphClient graph analysis, LLM dimension assessment with Qwen.
8 dimensions: Concurrency, Design Patterns, Messaging, API Design, Data Access, Reliability, Security, Testability.
Scoring: start at 10, deduct minor −0.5 / moderate −1.0 / major −2.0. Framework-weighted.
Bands: 8.5–10 Healthy · 6.0–8.4 Watch · <6.0 Critical.
Also: dead code (7 categories), MFE module-federation checks, shared-database detection, unused deps/imports/exports.

**Repository layout (47 repos):** React micro-frontends (module federation, migrating away from iframe) + Spring Boot microservices. Each MFE has a corresponding MS.
