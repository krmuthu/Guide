# OME Agent — Accuracy Enhancement Plan (Plan → Execute → Verify)

> **Problem:** Multi-pass produces correct individual changes but loses cross-file coherence. Quality improves with more context but context is bounded.
> **Root Cause:** Each pass sees its own files in full but has no global view. Pass 2 doesn’t see Pass 1’s files — only a carry-forward summary. Cross-file dependencies, shared types, interface contracts get lost between passes.
> **Solution:** Three-phase architecture — Plan (global view, cheap) → Execute (full files, per-pass) → Verify (catch errors, self-correct). The planning phase gives every execution pass a shared blueprint without needing all files in context.
> **Effort:** ~5 days

## 1. Why Multi-Pass Loses Accuracy

### The Five Accuracy Failure Modes

**1. Cross-file contract mismatch**
Pass 1 generates a new method signature in `OrderService.ts`. Pass 2 generates the caller in `OrderController.ts` — but uses the wrong parameter types because it only saw a carry-forward summary of Pass 1’s changes, not the actual generated signature.

**2. Missing shared type awareness**
`OrderDTO` is defined in a shared types file. Pass 1 adds a field to it. Pass 2 doesn’t know about the new field because the types file wasn’t in its context and the carry-forward only said “added field to OrderDTO” without the exact type definition.

**3. Import inconsistency**
Pass 1 imports from `@shared/utils`. Pass 2 imports the same utility from `../../shared/utils`. Both work but violate team conventions — the model couldn’t see both files to ensure consistency.

**4. Duplicate logic**
Pass 1 writes a validation helper inline in File A. Pass 2 writes the same validation inline in File B. With global view the model would extract it to a shared function. Without global view each pass solves its own problem locally.

**5. Merge conflicts in execution order**
Pass 1 says “change line 45 of shared.ts”. Pass 2 says “change line 48 of shared.ts”. These are adjacent and might conflict — but neither pass knew the other would touch this file because shared.ts was only referenced, not a target in either pass.

### Why “More Context” Fixes These

When everything fits in one call, the model sees all files simultaneously. It naturally:

- Uses consistent types across files
- Extracts shared logic
- Maintains import conventions
- Avoids touching the same lines from different angles

The challenge: replicate this global awareness without the global context.

## 2. Three-Phase Architecture

```
┌──────────────────────────────────────────────────────────────┐
│  PHASE 1: PLAN (fast model, single call)                     │
│                                                              │
│  Input:  Query + ticket + file SIGNATURES (not full files)   │
│          + graph context + conventions + similar PRs         │
│  Output: Implementation blueprint                            │
│          - Which files to change and how                     │
│          - Cross-file contracts (shared interfaces/types)    │
│          - Execution order                                   │
│          - What each file's changes depend on                │
│                                                              │
│  Context cost: ~6,000-10,000 tokens (signatures are small)   │
│  Model: Fast (:8090) — ~3-5 seconds                         │
└──────────────┬───────────────────────────────────────────────┘
               │ Blueprint (shared across all execution passes)
               ▼
┌──────────────────────────────────────────────────────────────┐
│  PHASE 2: EXECUTE (deep model, N passes)                     │
│                                                              │
│  Each pass receives:                                         │
│  - The BLUEPRINT from Phase 1 (global view)                  │
│  - Full target file(s) — UNCOMPRESSED                        │
│  - Cross-file interfaces extracted from other pass files     │
│  - Full ticket + conventions + graph                         │
│                                                              │
│  Each pass produces: CodeChange[] for its assigned files     │
│                                                              │
│  Model: Deep (:8091) — ~45-60s per pass                     │
└──────────────┬───────────────────────────────────────────────┘
               │ All CodeChange[] + NewFile[]
               ▼
┌──────────────────────────────────────────────────────────────┐
│  PHASE 3: VERIFY (fast model, 1-2 passes)                    │
│                                                              │
│  Input:  All generated changes + actual file content at      │
│          the changed lines + cross-file contracts            │
│  Checks:                                                     │
│          - Type compatibility across changes                 │
│          - Import consistency                                │
│          - No duplicate logic                                │
│          - Line numbers match actual file content            │
│          - Method signatures match between caller/callee     │
│                                                              │
│  Output: Verified changes OR correction instructions         │
│          If corrections needed → loop back to targeted       │
│          re-execution pass for specific files                │
│                                                              │
│  Model: Fast (:8090) — ~3-5 seconds                         │
└──────────────────────────────────────────────────────────────┘
```

### Why This Works

- **Phase 1 (PLAN)** gives global awareness cheaply. File signatures are 5-15 lines each instead of 200-400 lines. 8 files × 15 lines = 120 lines vs 8 files × 300 lines = 2,400 lines. The blueprint fits in one call even for large tasks.
- **Phase 2 (EXECUTE)** keeps full file fidelity. Each pass sees its target files uncompressed plus the blueprint plus cross-file interfaces from other passes. The model knows what to generate AND how it connects to changes in other files.
- **Phase 3 (VERIFY)** catches the errors that slip through. It’s cheap because it only needs to check contracts and consistency, not generate code.

### Single-Pass Optimization

When everything fits in one call, skip Phase 1 and Phase 3. The three-phase architecture only activates for multi-pass scenarios.

```
Fits in single call?
  YES → Generate directly (current behavior, no overhead)
  NO  → Plan → Execute (N passes) → Verify → (Correct if needed)
```

## 3. Phase 1 — PLAN (Global View, One Call)

### What Goes Into the Planning Prompt

|Section         |Content                                                                         |Token Cost      |
|----------------|--------------------------------------------------------------------------------|----------------|
|Query           |Full user request                                                               |~50-200         |
|Ticket          |Full ticket context (enriched)                                                  |~200-600        |
|File signatures |Imports + class declaration + method signatures for ALL target + reference files|~800-2,000      |
|Graph context   |Full callers, callees, blast radius                                             |~300-800        |
|Conventions     |Full convention output from fast model                                          |~200-400        |
|Similar PRs     |Full PR summaries                                                               |~400-900        |
|Cross-file types|Shared interfaces/types referenced by multiple files                            |~200-600        |
|**Total**       |                                                                                |**~2,200-5,500**|

This comfortably fits in one call to either model. Even with 8 target files + 4 reference files, signatures total ~2,000 tokens vs ~20,000 for full files.

### File Signature Extraction

```typescript
// src/guide/context/file_signatures.ts

import Parser from 'tree-sitter';

export interface FileSignature {
  filePath: string;
  language: string;
  imports: string;
  classDeclaration: string | null;
  decorators: string[];
  fields: string[];
  methodSignatures: MethodSignature[];
  exportedTypes: string[];
  totalLines: number;
}

export interface MethodSignature {
  name: string;
  fullSignature: string;
  decorators: string[];
  startLine: number;
  endLine: number;
  lineCount: number;
}

/**
 * Extract a compact signature from a file — everything EXCEPT method bodies.
 * This gives the model enough to understand the file's structure, contracts,
 * and public API without needing the full implementation.
 *
 * A 350-line service file compresses to ~30-50 lines of signatures.
 */
export function extractFileSignature(
  file: GatheredFile,
  parser: Parser
): FileSignature {
  const lines = file.content.split('\n');
  const tree = parser.parse(file.content);
  const root = tree.rootNode;

  // ── Imports (always full) ──
  const importLines = extractImportLines(root, lines);

  // ── Class/interface declaration ──
  const classDecl = extractClassDeclarationLine(root, lines);

  // ── Decorators on class ──
  const classDecorators = extractClassDecorators(root, lines);

  // ── Field declarations ──
  const fields = extractFieldLines(root, lines);

  // ── Method signatures (name + params + return type + decorators, no body) ──
  const methods = extractMethodSignatures(root, lines);

  // ── Exported types/interfaces ──
  const exportedTypes = extractExportedTypes(root, lines);

  return {
    filePath: file.filePath,
    language: detectLanguage(file.filePath),
    imports: importLines,
    classDeclaration: classDecl,
    decorators: classDecorators,
    fields,
    methodSignatures: methods,
    exportedTypes,
    totalLines: lines.length,
  };
}

/**
 * Format a signature for inclusion in the planning prompt.
 */
export function formatSignatureForPlan(sig: FileSignature): string {
  const parts: string[] = [];

  parts.push(`### ${sig.filePath} (${sig.totalLines} lines)`);
  parts.push('```' + sig.language);

  // Imports
  if (sig.imports) parts.push(sig.imports);

  // Exported types
  for (const t of sig.exportedTypes) {
    parts.push(t);
  }

  // Class declaration with decorators
  for (const d of sig.decorators) parts.push(d);
  if (sig.classDeclaration) parts.push(sig.classDeclaration);

  // Fields
  for (const f of sig.fields) parts.push(`  ${f}`);

  // Method signatures (one line each)
  for (const m of sig.methodSignatures) {
    const decs = m.decorators.map(d => `  ${d}`).join('\n');
    if (decs) parts.push(decs);
    parts.push(`  ${m.fullSignature}  // L${m.startLine}-${m.endLine} (${m.lineCount} lines)`);
  }

  parts.push('```');
  return parts.join('\n');
}


// ── AST Helpers ──

function extractImportLines(root: Parser.SyntaxNode, lines: string[]): string {
  const importNodes = root.children.filter(n =>
    n.type === 'import_statement' || n.type === 'import_declaration' ||
    n.type === 'package_declaration' ||
    (n.type === 'lexical_declaration' && lines[n.startPosition.row]?.includes('require('))
  );
  if (importNodes.length === 0) return '';

  const start = importNodes[0].startPosition.row;
  const end = importNodes[importNodes.length - 1].endPosition.row;
  return lines.slice(start, end + 1).join('\n');
}

function extractClassDeclarationLine(root: Parser.SyntaxNode, lines: string[]): string | null {
  const classNode = findFirstDescendant(root, [
    'class_declaration', 'class_definition', 'interface_declaration',
  ]);
  if (!classNode) return null;

  const row = classNode.startPosition.row;
  // Capture up to opening brace
  let endRow = row;
  for (let i = row; i < Math.min(row + 5, lines.length); i++) {
    endRow = i;
    if (lines[i]?.includes('{')) break;
  }
  return lines.slice(row, endRow + 1).join('\n');
}

function extractClassDecorators(root: Parser.SyntaxNode, lines: string[]): string[] {
  const decorators: string[] = [];
  const classNode = findFirstDescendant(root, ['class_declaration', 'class_definition']);
  if (!classNode) return decorators;

  // Look at siblings before the class declaration
  let sibling = classNode.previousNamedSibling;
  while (sibling && sibling.type === 'decorator') {
    decorators.unshift(lines[sibling.startPosition.row]?.trim() ?? '');
    sibling = sibling.previousNamedSibling;
  }
  return decorators;
}

function extractFieldLines(root: Parser.SyntaxNode, lines: string[]): string[] {
  const fields: string[] = [];
  const classBody = findFirstDescendant(root, ['class_body']);
  if (!classBody) return fields;

  for (const child of classBody.children) {
    if (
      child.type === 'field_definition' ||
      child.type === 'public_field_definition' ||
      child.type === 'property_declaration' ||
      (child.type === 'field_declaration' && !lines[child.startPosition.row]?.includes('('))
    ) {
      const line = lines.slice(child.startPosition.row, child.endPosition.row + 1).join(' ').trim();
      fields.push(line);
    }
  }
  return fields;
}

function extractMethodSignatures(root: Parser.SyntaxNode, lines: string[]): MethodSignature[] {
  const methods: MethodSignature[] = [];

  function walk(node: Parser.SyntaxNode) {
    const isMethod = [
      'method_definition', 'method_declaration',
      'function_declaration', 'function_definition',
    ].includes(node.type);

    if (isMethod) {
      const nameNode = node.childForFieldName('name');
      const name = nameNode?.text ?? 'anonymous';
      const startLine = node.startPosition.row + 1;
      const endLine = node.endPosition.row + 1;

      // Build signature: everything up to opening brace
      const firstLine = lines[node.startPosition.row]?.trim() ?? '';
      let sig = firstLine.replace(/\{[\s\S]*$/, '').trim();

      // If signature spans multiple lines (params list), capture them
      if (!sig.includes(')') && node.startPosition.row < node.endPosition.row) {
        for (let i = node.startPosition.row + 1; i <= Math.min(node.startPosition.row + 5, node.endPosition.row); i++) {
          sig += ' ' + (lines[i]?.trim() ?? '');
          if (sig.includes(')')) break;
        }
        sig = sig.replace(/\{[\s\S]*$/, '').trim();
      }

      // Preceding decorators
      const decorators: string[] = [];
      let prev = node.previousNamedSibling;
      while (prev && prev.type === 'decorator') {
        decorators.unshift(lines[prev.startPosition.row]?.trim() ?? '');
        prev = prev.previousNamedSibling;
      }

      methods.push({
        name,
        fullSignature: sig,
        decorators,
        startLine,
        endLine,
        lineCount: endLine - startLine + 1,
      });
      return; // Don't recurse into method bodies
    }

    for (const child of node.children) {
      walk(child);
    }
  }

  walk(root);
  return methods;
}

function extractExportedTypes(root: Parser.SyntaxNode, lines: string[]): string[] {
  const types: string[] = [];

  for (const child of root.children) {
    if (
      child.type === 'export_statement' ||
      child.type === 'interface_declaration' ||
      child.type === 'type_alias_declaration'
    ) {
      const text = lines.slice(child.startPosition.row, child.endPosition.row + 1).join('\n');
      // Only include if it's a type/interface (not a function/class export)
      if (/\b(interface|type)\b/.test(text)) {
        types.push(text);
      }
    }
  }

  return types;
}

function findFirstDescendant(node: Parser.SyntaxNode, types: string[]): Parser.SyntaxNode | null {
  if (types.includes(node.type)) return node;
  for (const child of node.children) {
    const found = findFirstDescendant(child, types);
    if (found) return found;
  }
  return null;
}

function detectLanguage(filePath: string): string {
  const ext = filePath.split('.').pop()?.toLowerCase();
  const map: Record<string, string> = {
    ts: 'typescript', tsx: 'typescript', js: 'javascript', jsx: 'javascript',
    java: 'java', kt: 'kotlin', py: 'python', go: 'go',
  };
  return map[ext ?? ''] ?? '';
}
```

// src/guide/context/plan_phase.ts

export function buildPlanPrompt(
  input: ContextInput,
  signatures: FileSignature[],
  crossFileTypes: string[],
): string {
  const sections: string[] = [];

  sections.push(`## Task
${input.query}`);

  // Ticket — full
  if (input.ticketContext) {
    sections.push(`## Ticket
${formatTicketFull(input.ticketContext)}`);
  }

  // Conventions — full
  if (input.conventions) {
    sections.push(`## Team Conventions
${input.conventions}`);
  }

  // All file signatures — compact but complete
  sections.push(`## Files Available for Modification`);
  for (const sig of signatures.filter(s => input.targetFiles.some(t => t.filePath === s.filePath))) {
    sections.push(formatSignatureForPlan(sig));
  }

  // Reference file signatures
  if (input.referenceFiles.length > 0) {
    sections.push(`## Reference Files (read-only, for pattern reference)`);
    for (const sig of signatures.filter(s => input.referenceFiles.some(r => r.filePath === s.filePath))) {
      sections.push(formatSignatureForPlan(sig));
    }
  }

  // Cross-file shared types
  if (crossFileTypes.length > 0) {
    sections.push(`## Shared Types (used across multiple files)
\`\`\`typescript
${crossFileTypes.join('\n\n')}
\`\`\``);
  }

  // Graph context — full
  sections.push(`## Dependencies & Impact
${formatGraphFull(input.graphContext)}`);

  // Similar PRs — full
  if (input.similarPRs.length > 0) {
    sections.push(`## Similar PRs
${formatPRsFull(input.similarPRs)}`);
  }

  // Planning instruction
  sections.push(`## Instructions

Create a detailed implementation BLUEPRINT. You will NOT write code now — a later phase will generate code for each file individually using this blueprint as guidance.

The blueprint must give each file's code generation pass enough information to produce changes that are COMPATIBLE with changes in other files.

Respond with JSON:
{
  "summary": "One paragraph overview of the full change",
  "fileChangePlans": [
    {
      "filePath": "exact/path.ts",
      "changeDescription": "What to change in this file and why",
      "methodsToModify": ["methodName"],
      "methodsToAdd": [
        {
          "name": "newMethodName",
          "signature": "async newMethodName(param: Type): Promise<ReturnType>",
          "purpose": "what this method does"
        }
      ],
      "importsToAdd": ["import { Thing } from '@module'"],
      "dependsOnFiles": ["other/file.ts"],
      "dependsOnChanges": ["description of what must exist in other files for this to work"],
      "contractsToHonor": [
        "Must accept OrderDTO with new 'status' field",
        "Must return same error format as existing submitOrder"
      ]
    }
  ],
  "newFiles": [
    {
      "path": "path/to/new.ts",
      "purpose": "why this new file is needed",
      "exports": ["what it exports"],
      "consumedBy": ["which existing files will import from it"]
    }
  ],
  "sharedContracts": [
    {
      "name": "OrderDTO",
      "change": "Add 'status: OrderStatus' field",
      "affectedFiles": ["order/types.ts", "order/service.ts", "order/controller.ts"]
    }
  ],
  "executionOrder": ["path/first.ts", "path/second.ts"],
  "crossFileRules": [
    "All error responses must use ErrorResponse from @shared/errors",
    "Import paths must use @ alias, not relative"
  ]
}`);

  return sections.join('\n\n---\n\n');
}

## 4. Phase 2 — EXECUTE (Full Files, Per-Pass)

Each execution pass now receives the **blueprint** from Phase 1 in addition to its full uncompressed files. This gives it global awareness without global context.

### What Each Execution Pass Sees

|Section              |Content                                                                  |Compressed?                     |
|---------------------|-------------------------------------------------------------------------|--------------------------------|
|Blueprint            |Full output from Phase 1                                                 |No — it’s ~800-1,500 tokens     |
|This file’s plan     |The specific `fileChangePlans` entry for this file                       |No                              |
|Cross-file rules     |From blueprint                                                           |No                              |
|Shared contracts     |Types/interfaces being modified across files                             |No                              |
|Cross-file interfaces|Signatures of methods in OTHER files that this file calls or is called by|Signatures only (not full files)|
|Target file(s)       |Full uncompressed with line numbers                                      |**No**                          |
|Reference files      |Full uncompressed                                                        |**No**                          |
|Ticket               |Full                                                                     |No                              |
|Conventions          |Full                                                                     |No                              |
|Carry-forward        |Prior pass changes (summary)                                             |Summary of model output only    |

### Updated Execution Pass Prompt

```typescript
// src/guide/context/execute_phase.ts

export function buildExecutePassPrompt(
  pass: PassDefinition,
  input: ContextInput,
  blueprint: ImplementationBlueprint,
  crossFileInterfaces: CrossFileInterface[],
  carryForward: CarryForwardEntry[]
): string {
  const sections: string[] = [];

  // ── System ──
  sections.push(getImplementSystemPrompt());

  // ── Task (from original query) ──
  sections.push(`## Task\n${input.query}`);

  // ── THE BLUEPRINT (global awareness) ──
  sections.push(formatBlueprintForExecution(blueprint, pass.targetFiles));

  // ── Cross-file rules ──
  if (blueprint.crossFileRules.length > 0) {
    sections.push(`## Cross-File Rules (apply to ALL changes)
${blueprint.crossFileRules.map(r => `- ${r}`).join('\n')}`);
  }

  // ── Shared contracts (exact type definitions being modified) ──
  if (blueprint.sharedContracts.length > 0) {
    const relevant = blueprint.sharedContracts.filter(sc =>
      sc.affectedFiles.some(f => pass.targetFiles.some(t => t.filePath === f))
    );
    if (relevant.length > 0) {
      sections.push(`## Shared Contracts (types modified across files)
${relevant.map(sc => `**${sc.name}**: ${sc.change}\nAffected: ${sc.affectedFiles.join(', ')}`).join('\n\n')}`);
    }
  }

  // ── Cross-file interfaces (signatures from OTHER pass files) ──
  if (crossFileInterfaces.length > 0) {
    sections.push(`## Interfaces from Other Files
These methods exist in files processed by other passes. Use these signatures for compatibility.

${crossFileInterfaces.map(cfi => `### ${cfi.filePath}
\`\`\`${cfi.language}
${cfi.signatures.join('\n')}
\`\`\``).join('\n\n')}`);
  }

  // ── Carry-forward from prior passes ──
  if (carryForward.length > 0) {
    sections.push(formatCarryForward(carryForward));
  }

  // ── Ticket (full) ──
  if (input.ticketContext) {
    sections.push(`## Ticket\n${formatTicketFull(input.ticketContext)}`);
  }

  // ── Conventions (full) ──
  if (input.conventions) {
    sections.push(`## Team Conventions\n${input.conventions}`);
  }

  // ── Target files (FULL, UNCOMPRESSED, with line numbers) ──
  sections.push('## Target Files — Generate changes for these');
  for (const file of pass.targetFiles) {
    sections.push(formatFileWithLineNumbers(file));
  }

  // ── Reference files (FULL, UNCOMPRESSED) ──
  if (pass.referenceFiles.length > 0) {
    sections.push('## Reference Files — Follow these patterns');
    for (const file of pass.referenceFiles) {
      sections.push(formatFileWithLineNumbers(file));
    }
  }

  // ── This file's specific plan ──
  const filePlans = blueprint.fileChangePlans.filter(fp =>
    pass.targetFiles.some(t => t.filePath === fp.filePath)
  );
  if (filePlans.length > 0) {
    sections.push(`## Your Specific Plan for This Pass
Follow this plan from the blueprint. Generate the exact changes described.

${filePlans.map(fp => `### ${fp.filePath}
- Change: ${fp.changeDescription}
- Modify methods: ${fp.methodsToModify.join(', ') || 'none'}
- Add methods: ${fp.methodsToAdd.map(m => `${m.name}: ${m.purpose}`).join('; ') || 'none'}
- Add imports: ${fp.importsToAdd.join(', ') || 'none'}
- Contracts: ${fp.contractsToHonor.join('; ') || 'none'}`).join('\n\n')}`);
  }

  // ── Output instructions ──
  sections.push(getExecuteOutputInstructions(pass));

  return sections.join('\n\n---\n\n');
}


/**
 * Format the blueprint for inclusion in an execution pass.
 * Show the FULL plan but highlight which files THIS pass handles.
 */
function formatBlueprintForExecution(
  blueprint: ImplementationBlueprint,
  thisPassFiles: GatheredFile[]
): string {
  const thisFilePaths = new Set(thisPassFiles.map(f => f.filePath));

  const fileList = blueprint.fileChangePlans.map(fp => {
    const isThisPass = thisFilePaths.has(fp.filePath);
    const marker = isThisPass ? '→ THIS PASS' : '  (other pass)';
    return `${marker} ${fp.filePath}: ${fp.changeDescription}`;
  }).join('\n');

  const newFileList = blueprint.newFiles.map(nf =>
    `  NEW: ${nf.path} — ${nf.purpose}`
  ).join('\n');

  return `## Implementation Blueprint (global plan)

${blueprint.summary}

### All Changes Planned
${fileList}
${newFileList ? '\n' + newFileList : ''}

### Execution Order
${blueprint.executionOrder.map((f, i) => `${i + 1}. ${f}`).join('\n')}`;
}

## 6. Cross-File Interface Extraction

When files are split across passes, extract the interfaces that connect them — method signatures that one file calls and another file defines.

```typescript
// src/guide/context/cross_file_interfaces.ts

export interface CrossFileInterface {
  filePath: string;
  language: string;
  /** Method signatures that other files call into or depend on */
  signatures: string[];
}

export interface CrossFileContract {
  callerFile: string;
  calleeFile: string;
  callerSignature: string;
  calleeSignature: string;
}

/**
 * For each pass, identify which OTHER pass's files are dependencies.
 * Extract just the method signatures from those files — enough for
 * type compatibility without full file content.
 *
 * Uses the knowledge graph to identify caller→callee relationships.
 */
export async function extractCrossFileInterfaces(
  passes: PassDefinition[],
  allFiles: GatheredFile[],
  graphContext: GraphGatherResult,
  parser: Parser,
): Promise<Map<number, CrossFileInterface[]>> {

  const result = new Map<number, CrossFileInterface[]>();

  // Build a map: filePath → which pass it's in
  const fileToPass = new Map<string, number>();
  for (const pass of passes) {
    if (pass.passType !== 'generate') continue;
    for (const file of pass.targetFiles) {
      fileToPass.set(file.filePath, pass.passNumber);
    }
  }

  // For each pass, find files in OTHER passes that THIS pass depends on
  for (const pass of passes) {
    if (pass.passType !== 'generate') continue;

    const crossInterfaces: CrossFileInterface[] = [];

    for (const targetFile of pass.targetFiles) {
      // Get callees and imports from graph
      const deps = getDependenciesFromGraph(targetFile.filePath, graphContext);

      for (const depFilePath of deps) {
        const depPass = fileToPass.get(depFilePath);

        // Only extract interface if the dep is in a DIFFERENT pass
        if (depPass !== undefined && depPass !== pass.passNumber) {
          const depFile = allFiles.find(f => f.filePath === depFilePath);
          if (!depFile) continue;

          const sig = extractFileSignature(depFile, parser);

          // Only include the methods that the target file actually calls
          const calledMethods = getCalledMethods(targetFile.filePath, depFilePath, graphContext);
          const relevantSigs = sig.methodSignatures
            .filter(m => calledMethods.includes(m.name))
            .map(m => {
              const decs = m.decorators.join('\n');
              return decs ? `${decs}\n${m.fullSignature}` : m.fullSignature;
            });

          if (relevantSigs.length > 0) {
            crossInterfaces.push({
              filePath: depFilePath,
              language: sig.language,
              signatures: relevantSigs,
            });
          }
        }
      }
    }

    result.set(pass.passNumber, crossInterfaces);
  }

  return result;
}


function getDependenciesFromGraph(
  filePath: string,
  graph: GraphGatherResult
): string[] {
  const deps: string[] = [];

  // From callees
  if (graph.callees) {
    for (const callee of graph.callees) {
      if (callee.filePath && callee.filePath !== filePath) {
        deps.push(callee.filePath);
      }
    }
  }

  // From imports
  if (graph.imports) {
    for (const imp of graph.imports) {
      if (imp.filePath && imp.filePath !== filePath) {
        deps.push(imp.filePath);
      }
    }
  }

  return [...new Set(deps)];
}

function getCalledMethods(
  callerFile: string,
  calleeFile: string,
  graph: GraphGatherResult
): string[] {
  const methods: string[] = [];

  if (graph.edges) {
    for (const edge of graph.edges) {
      if (
        edge.type === 'calls' &&
        edge.sourceFile === callerFile &&
        edge.targetFile === calleeFile
      ) {
        if (edge.targetName) methods.push(edge.targetName);
      }
    }
  }

  return [...new Set(methods)];
}

## 7. Dependency-Aware Pass Grouping

The current pass planner groups files by size (greedy bin packing). Better: group files that depend on each other into the SAME pass when possible.

```typescript
// src/guide/context/dependency_grouping.ts

/**
 * Group files into passes by dependency affinity — files that call each other
 * should be in the same pass when possible.
 *
 * Algorithm:
 * 1. Build dependency graph between target files
 * 2. Find connected components (files that transitively depend on each other)
 * 3. Try to fit each component into one pass
 * 4. If a component is too large, split at the weakest dependency edge
 */
export function groupFilesByDependency(
  files: GatheredFile[],
  fileTokens: Map<string, number>,
  graphContext: GraphGatherResult,
  maxTokensPerPass: number
): GatheredFile[][] {

  // ── Step 1: Build dependency adjacency between target files ──
  const fileSet = new Set(files.map(f => f.filePath));
  const edges: Array<[string, string, number]> = []; // [from, to, weight]

  for (const file of files) {
    const deps = getDependenciesFromGraph(file.filePath, graphContext);
    for (const dep of deps) {
      if (fileSet.has(dep) && dep !== file.filePath) {
        // Weight by number of call edges between them
        const callCount = countCallEdges(file.filePath, dep, graphContext);
        edges.push([file.filePath, dep, callCount]);
      }
    }
  }

  // ── Step 2: Find connected components ──
  const components = findConnectedComponents(files.map(f => f.filePath), edges);

  // ── Step 3: Fit components into passes ──
  const passes: GatheredFile[][] = [];

  for (const component of components) {
    const componentTokens = component.reduce(
      (sum, fp) => sum + (fileTokens.get(fp) ?? 0), 0
    );

    if (componentTokens <= maxTokensPerPass) {
      // Entire component fits in one pass — keep together
      passes.push(component.map(fp => files.find(f => f.filePath === fp)!));
    } else {
      // Component too large — split at weakest edge
      const subgroups = splitComponent(component, fileTokens, edges, maxTokensPerPass);
      for (const subgroup of subgroups) {
        passes.push(subgroup.map(fp => files.find(f => f.filePath === fp)!));
      }
    }
  }

  // ── Step 4: Add orphans (files with no dependencies on other targets) ──
  // These are already handled by connected components (singleton components)

  return passes;
}


function findConnectedComponents(
  nodes: string[],
  edges: Array<[string, string, number]>
): string[][] {
  const adj = new Map<string, Set<string>>();
  for (const node of nodes) adj.set(node, new Set());
  for (const [a, b] of edges) {
    adj.get(a)?.add(b);
    adj.get(b)?.add(a);
  }

  const visited = new Set<string>();
  const components: string[][] = [];

  for (const node of nodes) {
    if (visited.has(node)) continue;
    const component: string[] = [];
    const stack = [node];
    while (stack.length > 0) {
      const current = stack.pop()!;
      if (visited.has(current)) continue;
      visited.add(current);
      component.push(current);
      for (const neighbor of adj.get(current) ?? []) {
        if (!visited.has(neighbor)) stack.push(neighbor);
      }
    }
    components.push(component);
  }

  return components;
}


function splitComponent(
  component: string[],
  fileTokens: Map<string, number>,
  edges: Array<[string, string, number]>,
  maxTokensPerPass: number
): string[][] {
  // Greedy: sort files by token count, bin-pack into passes
  // Try to keep strongly connected files together
  const sorted = [...component].sort(
    (a, b) => (fileTokens.get(b) ?? 0) - (fileTokens.get(a) ?? 0)
  );

  const groups: string[][] = [[]];
  const groupTokens: number[] = [0];

  for (const file of sorted) {
    const tokens = fileTokens.get(file) ?? 0;

    // Find the group with most existing dependencies to this file
    let bestGroup = -1;
    let bestAffinity = -1;

    for (let i = 0; i < groups.length; i++) {
      if (groupTokens[i] + tokens > maxTokensPerPass) continue;

      const affinity = groups[i].reduce((sum, gf) => {
        const edgeWeight = edges.find(
          ([a, b]) => (a === file && b === gf) || (a === gf && b === file)
        )?.[2] ?? 0;
        return sum + edgeWeight;
      }, 0);

      if (affinity > bestAffinity) {
        bestAffinity = affinity;
        bestGroup = i;
      }
    }

    if (bestGroup >= 0) {
      groups[bestGroup].push(file);
      groupTokens[bestGroup] += tokens;
    } else {
      // Start new group
      groups.push([file]);
      groupTokens.push(tokens);
    }
  }

  return groups;
}


function countCallEdges(
  fileA: string, fileB: string, graph: GraphGatherResult
): number {
  let count = 0;
  if (graph.edges) {
    for (const edge of graph.edges) {
      if (edge.type === 'calls') {
        if (
          (edge.sourceFile === fileA && edge.targetFile === fileB) ||
          (edge.sourceFile === fileB && edge.targetFile === fileA)
        ) {
          count++;
        }
      }
    }
  }
  return count;
}

## 8. Verification Engine

Programmatic checks that run BEFORE the LLM verification pass — cheap and deterministic.

```typescript
// src/guide/context/verification_engine.ts

export interface VerificationResult {
  passed: boolean;
  issues: VerificationIssue[];
}

export interface VerificationIssue {
  type: 'line_mismatch' | 'overlapping_changes' | 'missing_import' |
        'missing_planned_change' | 'file_not_found';
  severity: 'error' | 'warning';
  file: string;
  changeId?: string;
  description: string;
  autoFixable: boolean;
  autoFix?: () => CodeChange;
}

/**
 * Run deterministic verification checks before the LLM verify pass.
 * Catches mechanical errors that don't need LLM reasoning.
 */
export async function runDeterministicChecks(
  allChanges: CodeChange[],
  allNewFiles: NewFile[],
  blueprint: ImplementationBlueprint,
  repoBasePath: string
): Promise<VerificationResult> {
  const issues: VerificationIssue[] = [];

  // ── Check 1: Line content matches actual file ──
  for (const change of allChanges) {
    if (!change.currentCode) continue;

    const filePath = path.join(repoBasePath, change.file);
    try {
      const content = await fs.readFile(filePath, 'utf-8');
      const lines = content.split('\n');
      const actualContent = lines
        .slice(change.currentCode.startLine - 1, change.currentCode.endLine)
        .join('\n');

      const similarity = computeSimilarity(
        normalizeWhitespace(actualContent),
        normalizeWhitespace(change.currentCode.content)
      );

      if (similarity < 0.75) {
        // Try to find the correct lines
        const correctedRange = findContentInFile(
          lines, change.currentCode.content
        );

        if (correctedRange) {
          issues.push({
            type: 'line_mismatch',
            severity: 'warning',
            file: change.file,
            changeId: change.id,
            description: `Line ${change.currentCode.startLine}-${change.currentCode.endLine} ` +
              `doesn't match. Content found at L${correctedRange.start}-${correctedRange.end}`,
            autoFixable: true,
            autoFix: () => ({
              ...change,
              currentCode: {
                ...change.currentCode!,
                startLine: correctedRange.start,
                endLine: correctedRange.end,
                content: lines.slice(correctedRange.start - 1, correctedRange.end).join('\n'),
              },
            }),
          });
        } else {
          issues.push({
            type: 'line_mismatch',
            severity: 'error',
            file: change.file,
            changeId: change.id,
            description: `Content at L${change.currentCode.startLine}-${change.currentCode.endLine} ` +
              `doesn't match and couldn't be found elsewhere in file`,
            autoFixable: false,
          });
        }
      }
    } catch {
      issues.push({
        type: 'file_not_found',
        severity: 'error',
        file: change.file,
        changeId: change.id,
        description: `File not found at ${filePath}`,
        autoFixable: false,
      });
    }
  }

  // ── Check 2: Overlapping line ranges in same file ──
  const changesByFile = groupBy(allChanges, c => c.file);
  for (const [filePath, changes] of Object.entries(changesByFile)) {
    for (let i = 0; i < changes.length; i++) {
      for (let j = i + 1; j < changes.length; j++) {
        const a = changes[i].currentCode;
        const b = changes[j].currentCode;
        if (!a || !b) continue;

        if (a.startLine <= b.endLine && b.startLine <= a.endLine) {
          issues.push({
            type: 'overlapping_changes',
            severity: 'error',
            file: filePath,
            changeId: `${changes[i].id} ↔ ${changes[j].id}`,
            description: `Changes overlap: ${changes[i].id} (L${a.startLine}-${a.endLine}) ` +
              `and ${changes[j].id} (L${b.startLine}-${b.endLine})`,
            autoFixable: false,
          });
        }
      }
    }
  }

  // ── Check 3: Blueprint adherence — every planned file was addressed ──
  for (const plan of blueprint.fileChangePlans) {
    const hasChanges = allChanges.some(c => c.file === plan.filePath);
    const hasNewFile = allNewFiles.some(f => f.path === plan.filePath);
    if (!hasChanges && !hasNewFile) {
      issues.push({
        type: 'missing_planned_change',
        severity: 'warning',
        file: plan.filePath,
        description: `Blueprint planned changes for ${plan.filePath} but no changes were generated`,
        autoFixable: false,
      });
    }
  }

  // ── Auto-fix what we can ──
  for (const issue of issues) {
    if (issue.autoFixable && issue.autoFix) {
      const fixed = issue.autoFix();
      const idx = allChanges.findIndex(c => c.id === issue.changeId);
      if (idx >= 0) {
        allChanges[idx] = fixed;
        issue.description += ' (auto-corrected)';
        issue.severity = 'warning';
      }
    }
  }

  return {
    passed: !issues.some(i => i.severity === 'error' && !i.autoFixable),
    issues,
  };
}


function findContentInFile(
  lines: string[],
  targetContent: string
): { start: number; end: number } | null {
  const targetLines = targetContent.split('\n');
  const targetFirst = normalizeWhitespace(targetLines[0]);

  for (let i = 0; i < lines.length; i++) {
    if (normalizeWhitespace(lines[i]) === targetFirst) {
      const candidate = lines.slice(i, i + targetLines.length).join('\n');
      if (computeSimilarity(
        normalizeWhitespace(candidate),
        normalizeWhitespace(targetContent)
      ) > 0.85) {
        return { start: i + 1, end: i + targetLines.length };
      }
    }
  }
  return null;
}

function normalizeWhitespace(s: string): string {
  return s.replace(/\s+/g, ' ').trim();
}

function computeSimilarity(a: string, b: string): number {
  if (a === b) return 1.0;
  const longer = a.length > b.length ? a : b;
  const shorter = a.length > b.length ? b : a;
  if (longer.length === 0) return 1.0;

  // Simple word overlap similarity
  const wordsA = new Set(a.split(/\s+/));
  const wordsB = new Set(b.split(/\s+/));
  let overlap = 0;
  for (const w of wordsA) if (wordsB.has(w)) overlap++;
  return overlap / Math.max(wordsA.size, wordsB.size);
}

function groupBy<T>(arr: T[], keyFn: (item: T) => string): Record<string, T[]> {
  const result: Record<string, T[]> = {};
  for (const item of arr) {
    const key = keyFn(item);
    (result[key] ??= []).push(item);
  }
  return result;
}

## 9. Self-Correction Loop

When verification finds errors, targeted re-execution fixes them without regenerating everything.

```typescript
// src/guide/context/correction_loop.ts

export interface CorrectionNeeded {
  file: string;
  changeId: string;
  issue: string;
  instruction: string;
}

/**
 * Re-execute ONLY the files that have issues.
 * The correction pass sees:
 * - The original file (full)
 * - The attempted change that failed verification
 * - The specific issue and correction instruction
 * - The blueprint (for global awareness)
 * - All OTHER changes that passed (so it can ensure compatibility)
 *
 * Max 2 correction loops to prevent infinite cycling.
 */
export async function runCorrectionLoop(
  corrections: CorrectionNeeded[],
  allChanges: CodeChange[],
  allNewFiles: NewFile[],
  input: ContextInput,
  blueprint: ImplementationBlueprint,
  ctx: AgentContext,
  maxAttempts: number = 2
): Promise<{ changes: CodeChange[]; newFiles: NewFile[]; attempts: number }> {

  let currentChanges = [...allChanges];
  let currentNewFiles = [...allNewFiles];
  let remainingCorrections = corrections;
  let attempt = 0;

  while (remainingCorrections.length > 0 && attempt < maxAttempts) {
    attempt++;
    console.log(
      `[Correction] Attempt ${attempt}: fixing ${remainingCorrections.length} issues`
    );

    // Group corrections by file
    const correctionsByFile = groupBy(remainingCorrections, c => c.file);

    for (const [filePath, fileCorrections] of Object.entries(correctionsByFile)) {
      const file = input.targetFiles.find(f => f.filePath === filePath)
        ?? input.referenceFiles.find(f => f.filePath === filePath);
      if (!file) continue;

      // Build correction prompt
      const prompt = buildCorrectionPrompt(
        file, fileCorrections, currentChanges, blueprint, input
      );

      const routing = routeToModel('implement', 'generate_code');
      const response = await ctx.llm.chat(prompt, routing);
      const parsed = parseImplementPassResponse(response.content);

      // Replace the failed changes with corrected ones
      for (const correctedChange of parsed.changes) {
        const idx = currentChanges.findIndex(c => c.id === correctedChange.id);
        if (idx >= 0) {
          currentChanges[idx] = correctedChange;
        } else {
          currentChanges.push(correctedChange);
        }
      }
    }

    // Re-verify
    const reVerify = await runDeterministicChecks(
      currentChanges, currentNewFiles, blueprint,
      input.repoBasePath ?? ''
    );

    if (reVerify.passed) {
      console.log(`[Correction] All issues resolved after ${attempt} attempt(s)`);
      return { changes: currentChanges, newFiles: currentNewFiles, attempts: attempt };
    }

    // Check if the LLM verification also passes
    remainingCorrections = reVerify.issues
      .filter(i => i.severity === 'error' && !i.autoFixable)
      .map(i => ({
        file: i.file,
        changeId: i.changeId ?? '',
        issue: i.description,
        instruction: `Fix: ${i.description}`,
      }));
  }

  if (remainingCorrections.length > 0) {
    console.warn(
      `[Correction] ${remainingCorrections.length} issues remain after ${maxAttempts} attempts. ` +
      `Proceeding with best-effort output.`
    );
  }

  return { changes: currentChanges, newFiles: currentNewFiles, attempts: attempt };
}


function buildCorrectionPrompt(
  file: GatheredFile,
  corrections: CorrectionNeeded[],
  allChanges: CodeChange[],
  blueprint: ImplementationBlueprint,
  input: ContextInput
): string {
  const sections: string[] = [];

  sections.push(`## Correction Task

Your previous code generation had issues. Fix ONLY the specific problems listed below.
Do NOT regenerate changes that passed verification.`);

  // Issues to fix
  sections.push(`## Issues to Fix
${corrections.map(c => `- **${c.changeId}**: ${c.issue}
  Instruction: ${c.instruction}`).join('\n\n')}`);

  // The original file (FULL)
  sections.push(`## File to Fix
${formatFileWithLineNumbers(file)}`);

  // The failed changes (so model can see what it generated wrong)
  const failedChangeIds = new Set(corrections.map(c => c.changeId));
  const failedChanges = allChanges.filter(c => failedChangeIds.has(c.id));
  sections.push(`## Your Previous Changes (with issues)
\`\`\`json
${JSON.stringify(failedChanges, null, 2)}
\`\`\``);

  // Other changes that PASSED (for cross-file compatibility)
  const passedChanges = allChanges.filter(c => !failedChangeIds.has(c.id));
  if (passedChanges.length > 0) {
    sections.push(`## Other Changes (passed verification — ensure compatibility)
${passedChanges.map(c =>
  `- ${c.id} [${c.action}] ${c.file}: ${c.description}` +
  (c.affectedSymbols.length > 0 ? ` (symbols: ${c.affectedSymbols.join(', ')})` : '')
).join('\n')}`);
  }

  // Blueprint for context
  sections.push(`## Blueprint Reference
${blueprint.summary}

Cross-file rules:
${blueprint.crossFileRules.map(r => `- ${r}`).join('\n')}`);

  // Conventions
  if (input.conventions) {
    sections.push(`## Conventions\n${input.conventions}`);
  }

  sections.push(`## Output
Return ONLY the corrected changes as JSON (same format as original).
Keep the same change IDs. Fix ONLY the issues listed above.`);

  return sections.join('\n\n---\n\n');
}

## 10. Integration with Existing Multi-Pass

### Updated Flow

```typescript
// src/guide/context/pass_orchestrator.ts — UPDATED

export async function executeWithPasses(
  input: ContextInput,
  mode: GuideMode,
  ctx: AgentContext
): Promise<GuideResponse> {

  const contextWindowSize = mode === 'understand'
    ? parseInt(process.env.FAST_MODEL_CTX_SIZE || '65536', 10)
    : parseInt(process.env.DEEP_MODEL_CTX_SIZE || '32768', 10);

  // ── Step 1: Estimate ──
  const estimate = estimateFullContext(input, mode, contextWindowSize);

  // ── Step 2: Single-pass shortcut ──
  if (estimate.fits) {
    console.log(`[Orchestrator] Single-pass — all content fits`);
    return executeSinglePass(input, mode, ctx);
  }

  // ═══ MULTI-PASS — Plan → Execute → Verify ═══
  console.log(
    `[Orchestrator] Multi-pass: ${estimate.totalRequired} tokens > ${contextWindowSize} window`
  );

  const parser = ctx.parser; // Reuse bootstrap parser pool

  // ── Step 3: Extract signatures for planning ──
  const allFiles = [...input.targetFiles, ...input.referenceFiles];
  const signatures = allFiles.map(f => extractFileSignature(f, parser));
  const crossFileTypes = extractSharedTypes(allFiles, parser);

  // ── Step 4: PHASE 1 — PLAN ──
  console.log(`[Orchestrator] Phase 1: Planning...`);
  const planPrompt = buildPlanPrompt(input, signatures, crossFileTypes);
  const planRouting = routeToModel(mode, 'plan');
  const planResponse = await ctx.llm.chat(planPrompt, planRouting);
  const blueprint = parseBlueprintResponse(planResponse.content);

  console.log(
    `[Orchestrator] Blueprint: ${blueprint.fileChangePlans.length} file plans, ` +
    `${blueprint.sharedContracts.length} contracts, ` +
    `${blueprint.crossFileRules.length} rules`
  );

  // ── Step 5: Group files by dependency ──
  const fileTokens = new Map(
    estimate.targetFiles.map(f => [f.filePath, f.tokens])
  );

  const sharedContextTokens = estimate.systemPrompt + estimate.query
    + estimate.ticket + estimate.conventions + estimate.graphContext
    + estimate.healthWarnings + estimate.knownFindings + estimate.errorContext
    + estimateTokens(JSON.stringify(blueprint)); // Blueprint included in every pass

  const maxFileTokensPerPass = contextWindowSize
    - sharedContextTokens
    - getGenerationReserve(mode)
    - 1_000; // Cross-file interfaces + carry-forward headroom

  const fileGroups = groupFilesByDependency(
    input.targetFiles, fileTokens, input.graphContext, maxFileTokensPerPass
  );

  console.log(
    `[Orchestrator] ${fileGroups.length} execution passes ` +
    `(dependency-grouped from ${input.targetFiles.length} files)`
  );

  // ── Step 6: Extract cross-file interfaces ──
  const passes = fileGroups.map((files, i) => ({
    passNumber: i + 1,
    passType: 'generate' as const,
    targetFiles: files,
    referenceFiles: input.referenceFiles, // Distributed by token budget below
    includeTicket: true,
    includePRs: i === 0,
    includeGraph: true,
    includeConventions: true,
    carryForward: i > 0,
    estimatedTokens: 0,
  }));

  const crossFileInterfaceMap = await extractCrossFileInterfaces(
    passes, allFiles, input.graphContext, parser
  );

  // ── Step 7: PHASE 2 — EXECUTE ──
  console.log(`[Orchestrator] Phase 2: Executing ${passes.length} passes...`);

  const allChanges: CodeChange[] = [];
  const allNewFiles: NewFile[] = [];
  const carryForward: CarryForwardEntry[] = [];

  for (const pass of passes) {
    console.log(
      `[Orchestrator] Pass ${pass.passNumber}/${passes.length}: ` +
      `${pass.targetFiles.map(f => f.filePath).join(', ')}`
    );

    const crossInterfaces = crossFileInterfaceMap.get(pass.passNumber) ?? [];

    const prompt = buildExecutePassPrompt(
      pass, input, blueprint, crossInterfaces, carryForward
    );

    const routing = routeToModel(mode, 'generate_code');
    const response = await ctx.llm.chat(prompt, routing);
    const parsed = parseImplementPassResponse(response.content);

    allChanges.push(...parsed.changes);
    if (parsed.newFiles) allNewFiles.push(...parsed.newFiles);

    // Build carry-forward
    carryForward.push({
      passNumber: pass.passNumber,
      filesProcessed: pass.targetFiles.map(f => f.filePath),
      changesSummary: parsed.changes.map(c => ({
        id: c.id,
        file: c.file,
        action: c.action,
        description: c.description,
        affectedSymbols: c.affectedSymbols,
        // NEW: Include actual method signatures for cross-pass compatibility
        proposedSignatures: extractSignaturesFromProposedCode(c.proposedCode),
      })),
      newFilesSummary: parsed.newFiles?.map(f => ({
        path: f.path,
        purpose: f.description,
        exports: extractExportsFromCode(f.content),
      })) ?? [],
      conventionsObserved: parsed.conventionsApplied ?? [],
    });
  }

  // ── Step 8: PHASE 3 — VERIFY ──
  console.log(`[Orchestrator] Phase 3: Verifying...`);

  // 8a. Deterministic checks (line numbers, overlaps, blueprint adherence)
  const deterministicResult = await runDeterministicChecks(
    allChanges, allNewFiles, blueprint, input.repoBasePath ?? ''
  );

  console.log(
    `[Orchestrator] Deterministic checks: ${deterministicResult.issues.length} issues ` +
    `(${deterministicResult.issues.filter(i => i.severity === 'error').length} errors)`
  );

  // 8b. LLM verification (type compatibility, import consistency, contract adherence)
  const crossFileContracts = buildCrossFileContracts(allChanges, input.graphContext);
  const verifyPrompt = buildVerifyPrompt(
    input, blueprint, allChanges, allNewFiles, crossFileContracts
  );
  const verifyRouting = routeToModel(mode, 'verify');
  const verifyResponse = await ctx.llm.chat(verifyPrompt, verifyRouting);
  const verification = parseVerificationResponse(verifyResponse.content);

  console.log(
    `[Orchestrator] LLM verification: ${verification.verdict} — ` +
    `${verification.issues.length} issues`
  );

  // 8c. Self-correction loop (if needed)
  let finalChanges = allChanges;
  let finalNewFiles = allNewFiles;
  let correctionAttempts = 0;

  if (verification.verdict === 'needs_correction') {
    const corrections: CorrectionNeeded[] = verification.issues
      .filter(i => i.severity === 'error')
      .map(i => ({
        file: i.file,
        changeId: i.changeId,
        issue: i.description,
        instruction: i.correction?.instruction ?? `Fix: ${i.description}`,
      }));

    if (corrections.length > 0) {
      console.log(`[Orchestrator] Running correction loop for ${corrections.length} issues...`);

      const corrected = await runCorrectionLoop(
        corrections, allChanges, allNewFiles,
        input, blueprint, ctx
      );

      finalChanges = corrected.changes;
      finalNewFiles = corrected.newFiles;
      correctionAttempts = corrected.attempts;
    }
  }

  // ── Step 9: Assemble final response ──
  const executionOrder = verification.executionOrder ?? blueprint.executionOrder;

  return assembleFinalResponse({
    summary: blueprint.summary,
    changes: finalChanges,
    newFiles: finalNewFiles,
    executionOrder,
    blueprint,
    verification,
    correctionAttempts,
    totalPasses: passes.length,
  });
}
