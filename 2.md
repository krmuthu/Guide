# OME Agent — Agentic Retrieval Loop

> **Problem:** The gather phase calls a fixed set of RAG tools blindly per mode. It fetches blast radius for a simple rename. It fetches similar PRs when no PRs are relevant. It fetches 8 files when 2 would suffice. Every unnecessary tool call wastes tokens in the final prompt and dilutes the model’s focus.
> **Solution:** Replace the static gather plan with an agentic retrieval loop. Start with semantic search. Let the fast model look at results and decide what to fetch next. Loop until context is sufficient. Stop early when enough is known.
> **Result:** Better accuracy (only relevant context), smaller prompts (no wasted tokens), faster responses (fewer unnecessary calls), and natural scaling (simple queries exit in 1-2 loops, complex queries loop more).
> **Effort:** ~5 days

## 1. Why Static Gathering Hurts Accuracy

### What Happens Today

```typescript
// Current: gather_planner.ts — static checklist per mode
if (mode === 'implement') {
  call search_semantic_code        // always
  call search_exact_match          // always if symbols found
  call find_symbol × N             // always per extracted symbol
  call get_code_context × 2        // always top 2 symbols
  call analyze_blast_radius        // always ← WASTEFUL for simple tasks
  call get_ticket_context          // always if ticket_id
  call get_project_info            // always
  call search_architecture         // always ← OFTEN IRRELEVANT
  call get_repo_config             // always
  call analyze_graph_health        // always ← RARELY NEEDED
  call suggest_related_files       // always if file hint
  call lookup_glossary             // always if terms found
}
```

### Three Ways This Hurts

**1. Token waste → reduced accuracy**
Every unnecessary result consumes tokens in the execution prompt. A blast radius analysis for “add a logging statement” takes ~500 tokens that could have been a second reference file. The model’s attention is finite — irrelevant context dilutes focus on what matters.

**2. Missing depth → incomplete answers**
Static plans go wide (many tools, shallow) instead of deep (fewer tools, follow the thread). When semantic search returns `AuthService.verifyToken`, the model should follow the call chain to discover `TokenValidator.checkExpiry` — but the static plan doesn’t know to do that. It called `get_code_context` on the top 2 symbols, which might be the wrong 2.

**3. Fixed tool set → blind spots**
The static plan can’t adapt. If search returns mostly config files, maybe it should fetch the actual service code. If search returns a controller, maybe it should find the service it delegates to. The current system can’t make these decisions.

### What Should Happen

```
Query: "Add pagination to bond search endpoint"

Loop 1: semantic_search("bond search endpoint pagination")
  → Found: BondSearchController.search(), BondSearchService.findBonds()
  → Decision: Need to see these files + understand current query params
  → Action: read_full_file(BondSearchController), get_code_context(findBonds)

Loop 2: read results
  → Found: Controller calls service.findBonds(criteria), returns List<BondDTO>
  → Decision: Need BondDTO type + existing pagination patterns in the codebase
  → Action: find_symbol(BondDTO), semantic_search("pagination Pageable Spring")

Loop 3: read results
  → Found: BondDTO in shared/types, other services use Spring Pageable
  → Decision: Have enough. Know the types, know the pattern, know the files.
  → Action: STOP — sufficient context

Total: 3 loops, 6 tool calls, all relevant
Static plan would have made: 12+ calls, half irrelevant
```

## 2. Agentic Loop Architecture

```
┌─────────────────────────────────────────────────────────────────────┐
│  ome_rag_guide receives query                                       │
│                                                                     │
│  ── Phase 0: SEED ──                                                │
│  Extract symbols, paths, ticket from query                          │
│  Perform initial semantic search (always — this is the seed)        │
│                                                                     │
│  ── Phase 1: AGENTIC RETRIEVAL LOOP ──                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │  LOOP (max N iterations):                                     │  │
│  │                                                               │  │
│  │  1. Fast model REASONS over accumulated context               │  │
│  │     - What do I know?                                         │  │
│  │     - What's missing?                                         │  │
│  │     - What should I fetch next?                               │  │
│  │     - OR: Do I have enough? → EXIT LOOP                      │  │
│  │                                                               │  │
│  │  2. Execute chosen retrieval actions (parallel)               │  │
│  │                                                               │  │
│  │  3. Accumulate results into context                           │  │
│  │                                                               │  │
│  │  4. Check sufficiency gate → continue or exit                 │  │
│  └───────────────────────────────────────────────────────────────┘  │
│                                                                     │
│  ── Phase 2: PLAN (if multi-pass) ──                                │
│  ── Phase 3: EXECUTE (deep model generates code) ──                 │
│  ── Phase 4: VERIFY ──                                              │
└─────────────────────────────────────────────────────────────────────┘
```

### Key Properties

- **Semantic search is always the first action** — the seed that everything grows from
- **The fast model decides** what to fetch next — not a static plan
- **Each loop iteration is one fast model call** (~1-2s) + parallel tool calls (~200-500ms)
- **The loop exits early** for simple queries (1-2 iterations) and goes deeper for complex ones (3-5)
- **Hard ceiling** prevents runaway loops (max 5 iterations, max 20 total tool calls)
- **Everything is parallel** within each iteration — multiple tool calls execute simultaneously
- **Results accumulate** — each iteration adds to the context, nothing is thrown away

## 3. Loop Mechanics

```
Iteration 0 (SEED — no LLM needed):
  ├── Extract query elements (symbols, file paths, ticket ID)
  ├── semantic_search(query)                          ← always
  ├── get_ticket_context(ticket_id) if present        ← always if ticket
  ├── lookup_glossary(terms) if found                 ← always if terms
  └── Accumulate seed results

Iteration 1..N (AGENTIC — fast model reasons):
  ├── Fast model receives:
  │   - Original query + mode
  │   - Accumulated context so far (search results, files, graph data)
  │   - List of available actions
  │   - Sufficiency criteria for this mode
  │
  ├── Fast model outputs:
  │   - Reasoning: "I found the controller but not the service it calls..."
  │   - Actions: [read_full_file("OrderService.java"), find_symbol("OrderDTO")]
  │   - OR: sufficient: true, reason: "Have all target files and patterns"
  │
  ├── Execute actions in parallel
  ├── Accumulate results
  └── Check hard limits (max iterations, max tools, max tokens accumulated)
```

### Why Fast Model for Reasoning

The retrieval reasoning call is small — ~1,000-2,000 tokens of accumulated context summaries + available actions. It produces ~100-200 tokens of output. The fast model (:8090) handles this in ~1-2 seconds. Using the deep model would be wasteful — this is a routing decision, not a generation task.

## 4. Retrieval Actions — The Toolbox

The fast model can choose from these actions. Each action maps to one or more RAG MCP tool calls.

```typescript
// src/guide/retrieval/actions.ts

export type RetrievalAction =
  | { action: 'semantic_search'; query: string; source?: 'code' | 'prose' }
  | { action: 'exact_search'; query: string }
  | { action: 'find_symbol'; symbol: string }
  | { action: 'read_full_file'; filePath: string }
  | { action: 'get_code_context'; symbol: string }
  | { action: 'get_callers'; symbol: string }
  | { action: 'get_callees'; symbol: string }
  | { action: 'get_blast_radius'; symbol: string }
  | { action: 'get_related_files'; filePath: string }
  | { action: 'get_recent_changes'; filePath: string }
  | { action: 'get_project_info'; repo: string }
  | { action: 'get_repo_config'; repo: string }
  | { action: 'search_similar_prs'; query: string }
  | { action: 'get_file_dependencies'; filePath: string }
  | { action: 'sufficient' }

export interface ActionResult {
  action: RetrievalAction;
  success: boolean;
  data: any;
  tokenEstimate: number;
  /** Summary for the reasoning model — compact, not full content */
  summary: string;
}

/**
 * Execute a retrieval action via RAG MCP tools.
 * Returns the result with a token estimate and compact summary.
 */
export async function executeAction(
  action: RetrievalAction,
  ctx: AgentContext
): Promise<ActionResult> {
  switch (action.action) {
    case 'semantic_search': {
      const results = await ctx.rag.searchCode(action.query, {
        source: action.source, topK: 10,
      });
      return {
        action, success: true,
        data: results,
        tokenEstimate: estimateTokens(JSON.stringify(results)),
        summary: `Found ${results.length} results: ${results.slice(0, 5).map(r =>
          `${r.filePath}:${r.symbolName ?? 'chunk'}(${r.score.toFixed(2)})`
        ).join(', ')}`,
      };
    }

    case 'read_full_file': {
      const content = await ctx.fileReader.readFile(action.filePath);
      const lines = content.split('\n').length;
      return {
        action, success: !!content,
        data: { filePath: action.filePath, content, lines },
        tokenEstimate: estimateTokens(content),
        summary: `Read ${action.filePath} (${lines} lines)`,
      };
    }

    case 'find_symbol': {
      const results = await ctx.rag.findSymbol(action.symbol);
      return {
        action, success: results.length > 0,
        data: results,
        tokenEstimate: estimateTokens(JSON.stringify(results)),
        summary: results.length > 0
          ? `Found ${action.symbol} in: ${results.map(r => `${r.filePath}:${r.startLine}`).join(', ')}`
          : `Symbol ${action.symbol} not found`,
      };
    }

    case 'get_code_context': {
      const ctx_result = await ctx.rag.getCodeContext(action.symbol);
      return {
        action, success: !!ctx_result,
        data: ctx_result,
        tokenEstimate: estimateTokens(JSON.stringify(ctx_result)),
        summary: `Context for ${action.symbol}: ${ctx_result?.callers?.length ?? 0} callers, ${ctx_result?.callees?.length ?? 0} callees`,
      };
    }

    case 'get_callers': {
      const callers = await ctx.rag.getCodeContext(action.symbol);
      return {
        action, success: !!callers,
        data: callers?.callers ?? [],
        tokenEstimate: estimateTokens(JSON.stringify(callers?.callers ?? [])),
        summary: `${action.symbol} called by: ${(callers?.callers ?? []).map((c: any) => c.name).join(', ') || 'none found'}`,
      };
    }

    case 'get_callees': {
      const callees = await ctx.rag.getCodeContext(action.symbol);
      return {
        action, success: !!callees,
        data: callees?.callees ?? [],
        tokenEstimate: estimateTokens(JSON.stringify(callees?.callees ?? [])),
        summary: `${action.symbol} calls: ${(callees?.callees ?? []).map((c: any) => c.name).join(', ') || 'none found'}`,
      };
    }

    case 'get_blast_radius': {
      const blast = await ctx.rag.analyzeBlastRadius(action.symbol);
      return {
        action, success: !!blast,
        data: blast,
        tokenEstimate: estimateTokens(JSON.stringify(blast)),
        summary: `Blast radius for ${action.symbol}: ${blast?.directDependents ?? 0} direct, ${blast?.transitiveDependents ?? 0} transitive`,
      };
    }

    case 'get_related_files': {
      const related = await ctx.rag.suggestRelatedFiles(action.filePath);
      return {
        action, success: related.length > 0,
        data: related,
        tokenEstimate: estimateTokens(JSON.stringify(related)),
        summary: `Related to ${action.filePath}: ${related.slice(0, 5).map((r: any) => r.filePath).join(', ')}`,
      };
    }

    case 'get_recent_changes': {
      const changes = await ctx.rag.getRecentChanges(action.filePath);
      return {
        action, success: changes.length > 0,
        data: changes,
        tokenEstimate: estimateTokens(JSON.stringify(changes)),
        summary: `${changes.length} recent changes to ${action.filePath}`,
      };
    }

    case 'get_project_info': {
      const info = await ctx.rag.getProjectInfo(action.repo);
      return {
        action, success: !!info,
        data: info,
        tokenEstimate: estimateTokens(JSON.stringify(info)),
        summary: `Project ${action.repo}: ${info?.techStack ?? 'unknown stack'}, ${info?.projectType ?? 'unknown type'}`,
      };
    }

    case 'get_repo_config': {
      const config = await ctx.rag.getRepoConfig(action.repo);
      return {
        action, success: !!config,
        data: config,
        tokenEstimate: estimateTokens(JSON.stringify(config)),
        summary: `Config for ${action.repo}: ${config ? 'loaded' : 'not found'}`,
      };
    }

    case 'search_similar_prs': {
      const prs = await searchSimilarPRs(action.query, ctx);
      return {
        action, success: prs.length > 0,
        data: prs,
        tokenEstimate: estimateTokens(JSON.stringify(prs)),
        summary: prs.length > 0
          ? `Found ${prs.length} similar PRs: ${prs.map(p => `PR-${p.pr_number}: ${p.title}`).join('; ')}`
          : 'No similar PRs found',
      };
    }

    case 'get_file_dependencies': {
      const deps = await ctx.rag.getFileDependencies(action.filePath);
      return {
        action, success: !!deps,
        data: deps,
        tokenEstimate: estimateTokens(JSON.stringify(deps)),
        summary: `${action.filePath} depends on: ${deps?.imports?.slice(0, 5).join(', ') ?? 'none'}`,
      };
    }

    case 'sufficient':
      return {
        action, success: true, data: null, tokenEstimate: 0,
        summary: 'Context sufficient — exiting retrieval loop',
      };
  }
}
```

## 5. Reasoning Prompt — The Brain

Each loop iteration, the fast model receives accumulated context summaries and decides what to fetch next.

```typescript
// src/guide/retrieval/reasoning_prompt.ts

export function buildReasoningPrompt(
  query: string,
  mode: GuideMode,
  accumulated: AccumulatedContext,
  availableActions: string[],
  iteration: number,
  maxIterations: number
): string {
  return `You are a retrieval planner for a code assistant. Your job is to decide what information to fetch next from the codebase to answer the developer's query.

## Developer Query
${query}

## Mode
${mode.toUpperCase()} — ${getModeDescription(mode)}

## What You Have So Far (Iteration ${iteration}/${maxIterations})

### Files Identified
${accumulated.files.length > 0
  ? accumulated.files.map(f =>
    `- ${f.filePath} (${f.lines} lines) — ${f.role}: ${f.relevance}`
  ).join('\n')
  : 'None yet'}

### Symbols Found
${accumulated.symbols.length > 0
  ? accumulated.symbols.map(s =>
    `- ${s.name} (${s.type}) in ${s.filePath}:${s.line}${s.context ? ' — ' + s.context : ''}`
  ).join('\n')
  : 'None yet'}

### Graph Context
${accumulated.graphSummary || 'Not fetched yet'}

### Search Results Summary
${accumulated.searchSummaries.join('\n') || 'No searches performed yet'}

### Ticket Context
${accumulated.ticketSummary || 'No ticket'}

### Similar PRs
${accumulated.prSummary || 'Not searched yet'}

### Project Info
${accumulated.projectInfo || 'Not fetched yet'}

## Token Budget
Accumulated context so far: ~${accumulated.totalTokens} tokens
Remaining budget for code generation: ~${accumulated.remainingBudget} tokens
${accumulated.remainingBudget < 5000 ? '⚠️ LOW BUDGET — only fetch what is essential' : ''}

## Available Actions
${availableActions.map(a => `- ${a}`).join('\n')}

## Sufficiency Criteria for ${mode.toUpperCase()} Mode
${getSufficiencyCriteria(mode)}

## Instructions

Think step by step:
1. What does the query need that I don't have yet?
2. Which of the available actions would fill that gap?
3. Is what I have already sufficient for ${mode} mode?

Respond with JSON:
{
  "reasoning": "One paragraph explaining what's missing and why you chose these actions",
  "sufficient": true/false,
  "actions": [
    { "action": "action_name", ...params },
    { "action": "action_name", ...params }
  ]
}

Rules:
- Maximum 4 actions per iteration (they execute in parallel)
- If sufficient is true, actions array must be empty
- Don't re-fetch something you already have
- Prefer specific actions (find_symbol, read_full_file) over broad ones (semantic_search) when you already know what you're looking for
- If search results show a function, get_code_context or get_callers/get_callees to understand how it connects — don't just read more files blindly
- For IMPLEMENT mode: you need target files (full), reference patterns, types, and conventions
- For UNDERSTAND mode: you need the code + its context (callers, callees, architecture)
- For DIAGNOSE mode: you need the error location, recent changes, and call chain to the error`;
}


function getModeDescription(mode: GuideMode): string {
  switch (mode) {
    case 'understand': return 'Explain code — entry points, flow, architecture. Need code + graph context.';
    case 'implement': return 'Generate code changes. Need target files, reference patterns, types, conventions.';
    case 'diagnose': return 'Find root cause + generate fix. Need error location, call chain, recent changes.';
  }
}

function getSufficiencyCriteria(mode: GuideMode): string {
  switch (mode) {
    case 'understand':
      return `Sufficient when:
- Found the relevant code (at least 1 file read or identified)
- Have caller/callee context for the main symbol
- Understand how it fits in the architecture
NOT sufficient when:
- Search returned results but haven't read any full files
- Found a function but don't know who calls it or what it calls`;

    case 'implement':
      return `Sufficient when:
- Identified ALL target files that need modification
- Read at least the primary target file(s) in full
- Found at least one reference file showing the team's patterns
- Know the types/interfaces involved (especially shared types)
- Have ticket context if a ticket was provided
NOT sufficient when:
- Don't know what types/DTOs the new code should use
- Haven't seen a reference implementation of a similar pattern
- Found a controller but haven't found the service it should call
- Missing shared types that multiple files reference`;

    case 'diagnose':
      return `Sufficient when:
- Found the file(s) where the error occurs
- Have the call chain leading to the error
- Have recent changes to affected files
- Understand the config context (if config-related)
NOT sufficient when:
- Have the error message but haven't found where in code it originates
- Found the error location but don't know what calls it
- Haven't checked recent changes that might have introduced the bug`;
  }
}
```

### Parse Reasoning Response

```typescript
// src/guide/retrieval/reasoning_parser.ts

export interface ReasoningOutput {
  reasoning: string;
  sufficient: boolean;
  actions: RetrievalAction[];
}

export function parseReasoningOutput(response: string): ReasoningOutput {
  try {
    // Extract JSON from response (handle markdown fences)
    const cleaned = response
      .replace(/```json\s*/g, '')
      .replace(/```\s*/g, '')
      .trim();

    const jsonMatch = cleaned.match(/\{[\s\S]*\}/);
    if (!jsonMatch) {
      return { reasoning: 'Failed to parse', sufficient: false, actions: [] };
    }

    const parsed = JSON.parse(jsonMatch[0]);

    return {
      reasoning: parsed.reasoning ?? '',
      sufficient: parsed.sufficient === true,
      actions: (parsed.actions ?? []).map(validateAction).filter(Boolean),
    };
  } catch {
    return { reasoning: 'Parse error', sufficient: false, actions: [] };
  }
}

function validateAction(raw: any): RetrievalAction | null {
  if (!raw?.action) return null;

  switch (raw.action) {
    case 'semantic_search':
      return raw.query ? { action: 'semantic_search', query: raw.query, source: raw.source } : null;
    case 'exact_search':
      return raw.query ? { action: 'exact_search', query: raw.query } : null;
    case 'find_symbol':
      return raw.symbol ? { action: 'find_symbol', symbol: raw.symbol } : null;
    case 'read_full_file':
      return raw.filePath ? { action: 'read_full_file', filePath: raw.filePath } : null;
    case 'get_code_context':
      return raw.symbol ? { action: 'get_code_context', symbol: raw.symbol } : null;
    case 'get_callers':
      return raw.symbol ? { action: 'get_callers', symbol: raw.symbol } : null;
    case 'get_callees':
      return raw.symbol ? { action: 'get_callees', symbol: raw.symbol } : null;
    case 'get_blast_radius':
      return raw.symbol ? { action: 'get_blast_radius', symbol: raw.symbol } : null;
    case 'get_related_files':
      return raw.filePath ? { action: 'get_related_files', filePath: raw.filePath } : null;
    case 'get_recent_changes':
      return raw.filePath ? { action: 'get_recent_changes', filePath: raw.filePath } : null;
    case 'get_project_info':
      return raw.repo ? { action: 'get_project_info', repo: raw.repo } : null;
    case 'get_repo_config':
      return raw.repo ? { action: 'get_repo_config', repo: raw.repo } : null;
    case 'search_similar_prs':
      return raw.query ? { action: 'search_similar_prs', query: raw.query } : null;
    case 'get_file_dependencies':
      return raw.filePath ? { action: 'get_file_dependencies', filePath: raw.filePath } : null;
    case 'sufficient':
      return { action: 'sufficient' };
    default:
      return null;
  }
}

## 6. Sufficiency Criteria — When to Stop

The loop exits when ANY of these conditions are met:

```typescript
// src/guide/retrieval/sufficiency.ts

export interface SufficiencyCheck {
  sufficient: boolean;
  reason: string;
}

/**
 * Hard gates — these override the model's judgment.
 */
export function checkHardLimits(
  accumulated: AccumulatedContext,
  iteration: number,
  config: RetrievalConfig
): SufficiencyCheck | null {

  // ── Max iterations reached ──
  if (iteration >= config.maxIterations) {
    return {
      sufficient: true,
      reason: `Max iterations reached (${config.maxIterations})`,
    };
  }

  // ── Max total tool calls reached ──
  if (accumulated.totalToolCalls >= config.maxToolCalls) {
    return {
      sufficient: true,
      reason: `Max tool calls reached (${config.maxToolCalls})`,
    };
  }

  // ── Token budget exhausted ──
  if (accumulated.remainingBudget < config.minRemainingBudget) {
    return {
      sufficient: true,
      reason: `Token budget nearly exhausted (${accumulated.remainingBudget} remaining)`,
    };
  }

  return null; // No hard limit hit — let the model decide
}

/**
 * Soft checks — the model should consider these but can override.
 * These are included in the reasoning prompt as guidance.
 */
export function getSoftSufficiencySignals(
  mode: GuideMode,
  accumulated: AccumulatedContext
): string[] {
  const signals: string[] = [];

  switch (mode) {
    case 'implement':
      if (accumulated.fullFilesRead === 0) {
        signals.push('NO_FULL_FILES: Haven\'t read any target files in full yet');
      }
      if (accumulated.fullFilesRead >= 2 && accumulated.hasConventionReference) {
        signals.push('LIKELY_SUFFICIENT: Have target files and convention reference');
      }
      if (!accumulated.hasTypes && accumulated.mentionedTypes.size > 0) {
        signals.push('MISSING_TYPES: Referenced types not yet fetched: ' +
          [...accumulated.mentionedTypes].join(', '));
      }
      break;

    case 'understand':
      if (accumulated.fullFilesRead >= 1 && accumulated.hasGraphContext) {
        signals.push('LIKELY_SUFFICIENT: Have code and graph context');
      }
      break;

    case 'diagnose':
      if (accumulated.fullFilesRead >= 1 && accumulated.hasRecentChanges) {
        signals.push('LIKELY_SUFFICIENT: Have error location and recent changes');
      }
      if (!accumulated.hasCallChain) {
        signals.push('MISSING_CALL_CHAIN: Haven\'t traced the call chain to the error');
      }
      break;
  }

  return signals;
}
```

## 7. Context Accumulator

Tracks everything gathered across iterations. Provides compact summaries for the reasoning prompt and full data for the execution phase.

```typescript
// src/guide/retrieval/context_accumulator.ts

export class ContextAccumulator {
  /** Full file contents — passed to execution phase */
  readonly files: Map<string, AccumulatedFile> = new Map();

  /** Symbols found — for tracking what's been discovered */
  readonly symbols: Map<string, SymbolInfo> = new Map();

  /** Search results — for dedup (don't search same query twice) */
  readonly searchQueries: Set<string> = new Set();
  readonly searchResults: SearchResult[] = [];

  /** Graph context — callers, callees, blast radius */
  graphData: GraphData | null = null;

  /** Ticket context */
  ticketContext: TicketContext | null = null;

  /** Similar PRs */
  similarPRs: PRResult[] = [];

  /** Project info */
  projectInfo: any = null;

  /** Repo config */
  repoConfig: any = null;

  /** Recent changes */
  recentChanges: RecentChange[] = [];

  /** Conventions (detected from reference files) */
  conventions: string | null = null;

  /** Track tool call count */
  totalToolCalls: number = 0;

  /** Track accumulated token estimate */
  totalTokens: number = 0;

  /** Context window budget */
  readonly contextBudget: number;

  /** Action history — what was fetched and when */
  readonly actionLog: ActionLogEntry[] = [];

  constructor(contextBudget: number) {
    this.contextBudget = contextBudget;
  }

  get remainingBudget(): number {
    return Math.max(0, this.contextBudget - this.totalTokens);
  }

  get fullFilesRead(): number {
    return [...this.files.values()].filter(f => f.fullContent !== null).length;
  }

  get hasGraphContext(): boolean {
    return this.graphData !== null;
  }

  get hasCallChain(): boolean {
    return (this.graphData?.callers?.length ?? 0) > 0 ||
           (this.graphData?.callees?.length ?? 0) > 0;
  }

  get hasRecentChanges(): boolean {
    return this.recentChanges.length > 0;
  }

  get hasConventionReference(): boolean {
    return [...this.files.values()].some(f => f.role === 'reference');
  }

  get hasTypes(): boolean {
    return [...this.symbols.values()].some(s =>
      s.type === 'interface' || s.type === 'type'
    );
  }

  get mentionedTypes(): Set<string> {
    // Types referenced in search results or file content but not yet resolved
    const mentioned = new Set<string>();
    for (const result of this.searchResults) {
      // Extract type references from search result content
      const typeRefs = extractTypeReferences(result.content);
      for (const t of typeRefs) {
        if (!this.symbols.has(t)) mentioned.add(t);
      }
    }
    return mentioned;
  }

  /** Absorb an action result into the accumulator */
  absorb(result: ActionResult): void {
    this.totalToolCalls++;
    this.actionLog.push({
      iteration: this.actionLog.length,
      action: result.action,
      success: result.success,
      summary: result.summary,
      tokensAdded: result.tokenEstimate,
    });

    if (!result.success) return;

    switch (result.action.action) {
      case 'semantic_search':
      case 'exact_search':
        this.searchQueries.add(
          result.action.action === 'semantic_search'
            ? result.action.query
            : (result.action as any).query
        );
        for (const r of result.data ?? []) {
          // Track file paths and symbols from search results
          if (r.filePath && !this.files.has(r.filePath)) {
            this.files.set(r.filePath, {
              filePath: r.filePath,
              fullContent: null, // Not read yet
              lines: 0,
              role: 'candidate', // Upgrade to 'target' or 'reference' when read
              relevance: `search score ${r.score?.toFixed(2)}`,
              fromSearch: true,
            });
          }
          if (r.symbolName) {
            this.symbols.set(r.symbolName, {
              name: r.symbolName,
              type: r.symbolType ?? 'unknown',
              filePath: r.filePath,
              line: r.startLine,
              context: r.graphContext?.centralityLabel,
            });
          }
          this.searchResults.push(r);
        }
        this.totalTokens += result.tokenEstimate;
        break;

      case 'read_full_file': {
        const fileData = result.data;
        const existing = this.files.get(fileData.filePath);
        this.files.set(fileData.filePath, {
          filePath: fileData.filePath,
          fullContent: fileData.content,
          lines: fileData.lines,
          role: existing?.role === 'candidate' ? 'target' : (existing?.role ?? 'target'),
          relevance: existing?.relevance ?? 'directly requested',
          fromSearch: existing?.fromSearch ?? false,
        });
        this.totalTokens += result.tokenEstimate;
        break;
      }

      case 'find_symbol':
        for (const s of result.data ?? []) {
          this.symbols.set(s.name ?? result.action.symbol, {
            name: s.name ?? result.action.symbol,
            type: s.nodeType ?? 'unknown',
            filePath: s.filePath,
            line: s.startLine,
            context: s.signature,
          });
        }
        this.totalTokens += result.tokenEstimate;
        break;

      case 'get_code_context':
      case 'get_callers':
      case 'get_callees':
        this.graphData = this.graphData ?? {};
        if (result.data?.callers) {
          this.graphData.callers = [
            ...(this.graphData.callers ?? []),
            ...result.data.callers,
          ];
        }
        if (result.data?.callees) {
          this.graphData.callees = [
            ...(this.graphData.callees ?? []),
            ...result.data.callees,
          ];
        }
        this.totalTokens += result.tokenEstimate;
        break;

      case 'get_blast_radius':
        this.graphData = this.graphData ?? {};
        this.graphData.blastRadius = result.data;
        this.totalTokens += result.tokenEstimate;
        break;

      case 'search_similar_prs':
        this.similarPRs = result.data ?? [];
        this.totalTokens += result.tokenEstimate;
        break;

      case 'get_recent_changes':
        this.recentChanges.push(...(result.data ?? []));
        this.totalTokens += result.tokenEstimate;
        break;

      case 'get_project_info':
        this.projectInfo = result.data;
        this.totalTokens += result.tokenEstimate;
        break;

      case 'get_repo_config':
        this.repoConfig = result.data;
        this.totalTokens += result.tokenEstimate;
        break;

      default:
        this.totalTokens += result.tokenEstimate;
    }
  }

  /** Check if an action would be redundant */
  isRedundant(action: RetrievalAction): boolean {
    switch (action.action) {
      case 'semantic_search':
        return this.searchQueries.has(action.query);
      case 'read_full_file':
        return this.files.get(action.filePath)?.fullContent !== null;
      case 'find_symbol':
        return this.symbols.has(action.symbol);
      case 'get_project_info':
        return this.projectInfo !== null;
      case 'get_repo_config':
        return this.repoConfig !== null;
      default:
        return false;
    }
  }

  /** Generate compact summary for the reasoning prompt */
  toReasoningSummary(): AccumulatedContextSummary {
    return {
      files: [...this.files.values()].map(f => ({
        filePath: f.filePath,
        lines: f.lines,
        role: f.role,
        relevance: f.relevance,
        hasFullContent: f.fullContent !== null,
      })),
      symbols: [...this.symbols.values()],
      searchSummaries: this.actionLog
        .filter(a => a.action.action === 'semantic_search' || a.action.action === 'exact_search')
        .map(a => a.summary),
      graphSummary: this.graphData ? formatGraphSummary(this.graphData) : null,
      ticketSummary: this.ticketContext
        ? `${this.ticketContext.ticketId}: ${this.ticketContext.summary}`
        : null,
      prSummary: this.similarPRs.length > 0
        ? this.similarPRs.map(p => `PR-${p.pr_number}: ${p.title}`).join('; ')
        : null,
      projectInfo: this.projectInfo
        ? `${this.projectInfo.techStack}, ${this.projectInfo.projectType}`
        : null,
      totalTokens: this.totalTokens,
      remainingBudget: this.remainingBudget,
    };
  }

  /**
   * Convert accumulated context to execution input.
   * This produces the full, uncompressed context for code generation.
   */
  toExecutionInput(query: string, mode: GuideMode): ContextInput {
    const targetFiles: GatheredFile[] = [];
    const referenceFiles: GatheredFile[] = [];

    for (const [filePath, file] of this.files) {
      if (!file.fullContent) continue;

      const gathered: GatheredFile = {
        filePath,
        content: file.fullContent,
        lines: file.lines,
        isTarget: file.role === 'target',
      };

      if (file.role === 'target') {
        targetFiles.push(gathered);
      } else if (file.role === 'reference') {
        referenceFiles.push(gathered);
      }
    }

    return {
      query,
      targetFiles,
      referenceFiles,
      ticketContext: this.ticketContext,
      similarPRs: this.similarPRs,
      graphContext: this.graphData ?? {},
      conventions: this.conventions,
      healthWarnings: [],
      knownFindings: [],
      recentChanges: this.recentChanges,
      repoBasePath: '', // Set by orchestrator
    };
  }
}


function extractTypeReferences(content: string): string[] {
  // Extract PascalCase identifiers that look like type names
  const matches = content.match(/\b[A-Z][a-zA-Z0-9]*(?:DTO|Entity|Request|Response|Type|Interface|Model|Config|Service|Repository|Controller)\b/g);
  return [...new Set(matches ?? [])];
}

function formatGraphSummary(graph: GraphData): string {
  const parts: string[] = [];
  if (graph.callers?.length) {
    parts.push(`Callers: ${graph.callers.slice(0, 5).map((c: any) => c.name).join(', ')}`);
  }
  if (graph.callees?.length) {
    parts.push(`Callees: ${graph.callees.slice(0, 5).map((c: any) => c.name).join(', ')}`);
  }
  if (graph.blastRadius) {
    parts.push(`Blast: ${graph.blastRadius.directDependents} direct deps`);
  }
  return parts.join(' | ') || 'Partial graph data';
}

## 8. Loop Controller

The main loop that coordinates seed → reason → execute → accumulate → check.

```typescript
// src/guide/retrieval/loop_controller.ts

import pLimit from 'p-limit';

export interface RetrievalLoopResult {
  context: ContextInput;
  iterations: number;
  totalToolCalls: number;
  totalReasoningCalls: number;
  exitReason: string;
  actionLog: ActionLogEntry[];
}

export async function runRetrievalLoop(
  query: string,
  mode: GuideMode,
  params: GuideParams,
  ctx: AgentContext,
  config: RetrievalConfig
): Promise<RetrievalLoopResult> {
  const startMs = performance.now();

  const contextWindowSize = mode === 'understand'
    ? config.fastModelCtxSize
    : config.deepModelCtxSize;

  const generationReserve = getGenerationReserve(mode);
  const contextBudget = contextWindowSize - generationReserve - 1_500; // system prompt + overhead

  const accumulator = new ContextAccumulator(contextBudget);
  const actionParallel = pLimit(config.maxParallelActions);

  // ── SEED: Initial actions that always happen ──
  const seedActions = buildSeedActions(query, mode, params);

  console.log(
    `[RetrievalLoop] Seed: ${seedActions.length} actions for "${query.slice(0, 80)}..."`
  );

  const seedResults = await Promise.allSettled(
    seedActions.map(a => actionParallel(() => executeAction(a, ctx)))
  );

  for (const result of seedResults) {
    if (result.status === 'fulfilled') {
      accumulator.absorb(result.value);
    }
  }

  console.log(
    `[RetrievalLoop] After seed: ${accumulator.files.size} files, ` +
    `${accumulator.symbols.size} symbols, ~${accumulator.totalTokens} tokens`
  );

  // ── AGENTIC LOOP ──
  let iteration = 0;
  let exitReason = '';
  let totalReasoningCalls = 0;

  while (iteration < config.maxIterations) {
    iteration++;

    // Check hard limits
    const hardLimit = checkHardLimits(accumulator, iteration, config);
    if (hardLimit) {
      exitReason = hardLimit.reason;
      console.log(`[RetrievalLoop] Hard limit: ${exitReason}`);
      break;
    }

    // Build reasoning prompt
    const summary = accumulator.toReasoningSummary();
    const softSignals = getSoftSufficiencySignals(mode, accumulator);
    const availableActions = getAvailableActions(mode, accumulator);

    const reasoningPrompt = buildReasoningPrompt(
      query, mode, summary, availableActions, iteration, config.maxIterations
    );

    // Add soft signals as hints
    const fullPrompt = softSignals.length > 0
      ? reasoningPrompt + `\n\n## Sufficiency Signals\n${softSignals.join('\n')}`
      : reasoningPrompt;

    // Ask fast model to reason
    const routing = routeToModel(mode, 'retrieval_reason');
    const reasoningResponse = await ctx.llm.chat(fullPrompt, routing);
    totalReasoningCalls++;

    const reasoning = parseReasoningOutput(reasoningResponse.content);

    console.log(
      `[RetrievalLoop] Iteration ${iteration}: ${reasoning.reasoning.slice(0, 120)}... ` +
      `→ ${reasoning.sufficient ? 'SUFFICIENT' : `${reasoning.actions.length} actions`}`
    );

    // Check if model says sufficient
    if (reasoning.sufficient) {
      exitReason = `Model determined sufficient: ${reasoning.reasoning}`;
      break;
    }

    // Filter redundant actions
    const newActions = reasoning.actions.filter(a => {
      if (accumulator.isRedundant(a)) {
        console.log(`[RetrievalLoop] Skipping redundant: ${a.action}`);
        return false;
      }
      return true;
    });

    if (newActions.length === 0) {
      exitReason = 'No new actions to take — all suggestions redundant';
      break;
    }

    // Cap actions per iteration
    const cappedActions = newActions.slice(0, config.maxActionsPerIteration);

    // Execute in parallel
    const results = await Promise.allSettled(
      cappedActions.map(a => actionParallel(() => executeAction(a, ctx)))
    );

    let newTokens = 0;
    for (const result of results) {
      if (result.status === 'fulfilled') {
        accumulator.absorb(result.value);
        newTokens += result.value.tokenEstimate;
      } else {
        console.warn(`[RetrievalLoop] Action failed:`, result.reason);
      }
    }

    console.log(
      `[RetrievalLoop] After iteration ${iteration}: ` +
      `${accumulator.files.size} files (${accumulator.fullFilesRead} read), ` +
      `${accumulator.symbols.size} symbols, ` +
      `~${accumulator.totalTokens} tokens (+${newTokens})`
    );
  }

  if (!exitReason) {
    exitReason = 'Max iterations reached';
  }

  // ── POST-LOOP: Convention detection ──
  if (mode === 'implement' && accumulator.fullFilesRead > 0) {
    const referenceFiles = [...accumulator.files.values()]
      .filter(f => f.role === 'reference' && f.fullContent);

    if (referenceFiles.length > 0) {
      const conventionPrompt = buildConventionPrompt(referenceFiles);
      const convRouting = routeToModel(mode, 'convention_detect');
      const convResponse = await ctx.llm.chat(conventionPrompt, convRouting);
      accumulator.conventions = convResponse.content;
    }
  }

  // ── Classify file roles ──
  classifyFileRoles(accumulator, mode, query);

  const totalMs = Math.round(performance.now() - startMs);
  console.log(
    `[RetrievalLoop] Complete: ${iteration} iterations, ` +
    `${accumulator.totalToolCalls} tool calls, ` +
    `${totalReasoningCalls} reasoning calls, ` +
    `${totalMs}ms. Exit: ${exitReason}`
  );

  return {
    context: accumulator.toExecutionInput(query, mode),
    iterations: iteration,
    totalToolCalls: accumulator.totalToolCalls,
    totalReasoningCalls,
    exitReason,
    actionLog: accumulator.actionLog,
  };
}


/**
 * Seed actions — always executed before the loop starts.
 */
function buildSeedActions(
  query: string,
  mode: GuideMode,
  params: GuideParams
): RetrievalAction[] {
  const actions: RetrievalAction[] = [];

  // Always: semantic search
  actions.push({ action: 'semantic_search', query });

  // If ticket provided: get ticket context
  if (params.ticket_id) {
    // Ticket context is fetched via getTicketContext which is not in our action list
    // It's handled separately in the seed phase
  }

  // If current file provided: read it
  if (params.current_file) {
    actions.push({ action: 'read_full_file', filePath: params.current_file });
  }

  // If current repo provided: get project info
  if (params.current_repo) {
    actions.push({ action: 'get_project_info', repo: params.current_repo });
  }

  // Extract symbols from query and search for them
  const symbols = extractSymbolsFromQuery(query);
  for (const symbol of symbols.slice(0, 2)) {
    actions.push({ action: 'find_symbol', symbol });
  }

  return actions;
}


/**
 * Classify accumulated files as target or reference based on
 * search relevance, mode, and query.
 */
function classifyFileRoles(
  accumulator: ContextAccumulator,
  mode: GuideMode,
  query: string
): void {
  const files = [...accumulator.files.values()].filter(f => f.fullContent);

  if (mode === 'implement') {
    // Target: files most relevant to the query (top by search score)
    // Reference: files that show patterns to follow
    const sorted = files.sort((a, b) => {
      const scoreA = accumulator.searchResults
        .filter(r => r.filePath === a.filePath)
        .reduce((max, r) => Math.max(max, r.score ?? 0), 0);
      const scoreB = accumulator.searchResults
        .filter(r => r.filePath === b.filePath)
        .reduce((max, r) => Math.max(max, r.score ?? 0), 0);
      return scoreB - scoreA;
    });

    // Top files are targets, rest are references
    const maxTargets = 4;
    for (let i = 0; i < sorted.length; i++) {
      sorted[i].role = i < maxTargets ? 'target' : 'reference';
    }
  } else {
    // UNDERSTAND / DIAGNOSE: all files are targets (being analyzed)
    for (const file of files) {
      file.role = 'target';
    }
  }
}


function getAvailableActions(mode: GuideMode, acc: ContextAccumulator): string[] {
  const actions = [
    'semantic_search { query: "search text", source?: "code"|"prose" } — find relevant code/docs',
    'exact_search { query: "identifier" } — exact match for function/class names',
    'find_symbol { symbol: "SymbolName" } — locate a specific symbol in the codebase',
    'read_full_file { filePath: "path/to/file" } — read a file\'s full content',
    'get_code_context { symbol: "name" } — get callers, callees, centrality for a symbol',
    'get_callers { symbol: "name" } — who calls this symbol',
    'get_callees { symbol: "name" } — what does this symbol call',
    'get_related_files { filePath: "path" } — files that often change with this one',
    'get_file_dependencies { filePath: "path" } — imports/dependencies of a file',
    'sufficient — stop gathering, context is complete',
  ];

  if (mode === 'implement') {
    actions.push(
      'get_blast_radius { symbol: "name" } — impact analysis for changes',
      'search_similar_prs { query: "what was done" } — find past PRs with similar changes',
    );
  }

  if (mode === 'diagnose') {
    actions.push(
      'get_recent_changes { filePath: "path" } — recent git changes to a file',
    );
  }

  if (!acc.projectInfo) {
    actions.push('get_project_info { repo: "repo-slug" } — tech stack and project type');
  }

  if (!acc.repoConfig) {
    actions.push('get_repo_config { repo: "repo-slug" } — application config');
  }

  return actions;
}


function extractSymbolsFromQuery(query: string): string[] {
  // Extract PascalCase, camelCase, and UPPER_CASE identifiers

  return [...new Set(matches ?? [])];
}

## 11. Loop Optimizations

### Parallel Action Execution

All actions in a single iteration execute in parallel. The model requests up to 4 actions per iteration — they all fire simultaneously.

```typescript
const results = await Promise.allSettled(
  actions.map(a => actionParallel(() => executeAction(a, ctx)))
);
```

### Dedup Guard

The accumulator tracks what’s been fetched. If the reasoning model asks for something already retrieved, it’s skipped without counting against the iteration budget.

### Token-Aware Exit

The accumulator continuously tracks estimated tokens. When the remaining budget drops below a threshold, the loop exits regardless of sufficiency — preventing the execution phase from hitting context limits.

### Search Query Rewriting

If the first semantic search returns poor results (all scores < 0.3), the reasoning model naturally suggests a rephrased search in the next iteration — without needing special logic.

### File Role Promotion

When a file starts as a `candidate` (found via search) and gets read via `read_full_file`, it’s promoted to `target` or `reference`. This prevents the execution phase from receiving unread file stubs.

### Expected Behavior by Query Complexity

|Query                                   |Iterations|Tool Calls|Why                                                   |
|----------------------------------------|----------|----------|------------------------------------------------------|

|“Add logging to OrderService”           |2-3       |5-8       |Read target + find logging pattern reference          |
|“Implement BMT-1234: Add pagination”    |3-4       |8-14      |Target files + types + patterns + ticket + similar PRs|

### Latency Impact

|Phase                  |Static Gather|Agentic Loop (simple) |Agentic Loop (complex)           |
|-----------------------|-------------|----------------------|---------------------------------|
|Gather/Retrieve        |~2-3s (fixed)|~3-4s (1-2 iterations)|~6-10s (3-5 iterations)          |
|Reasoning overhead     |0            |~1-2s per iteration   |~5-10s total                     |
|Wasted tokens in prompt|~3,000-5,000 |~0-500                |~0-500                           |
|Total code generation  |~50s         |~45s (tighter prompt) |~50s                             |
|**Net effect**         |             |**Faster + better**   |**Slightly slower + much better**|

The reasoning calls (~1-2s each) are paid back by smaller, more focused execution prompts that generate more accurate code on the first try — saving correction loops.
