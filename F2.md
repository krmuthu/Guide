## 1. Executive Summary

### 1.1 Problem

We converted HP NonStop (Tandem) COBOL applications to Java using Windsurf. We need auditable, fool-proof evidence that every COBOL construct has been faithfully converted — covering structural mapping, dependency integrity, and behavioral I/O parity.

### 1.2 Solution

Project Sentinel provides three primary verification layers plus an optional fourth, each catching what the previous layers miss:

|Layer       |Weight      |Required     |Method                                |Catches                                                  |
|------------|------------|-------------|--------------------------------------|---------------------------------------------------------|
|Structural  |25%         |Yes          |ProLeap ASG + name/pattern matching   |Direct mappings, naming convention matches               |
|Dependency  |30%         |Yes          |Postgres recursive CTEs on graph_edges|Broken call chains, missing dependencies, data flow gaps |
|Behavioral  |45%         |Yes          |Golden file parity testing            |Logic errors, edge case failures, numeric precision drift|
|**Semantic**|**optional**|**No (flag)**|**Dual-model embeddings + pgvector**  |**Refactored logic, renamed constructs, fuzzy matches**  |

Weights when semantic layer is disabled (default): Structural 25% + Dependency 30% + Behavioral 45% = 100%.

Weights when semantic layer is enabled: Structural 20% + Semantic 20% + Dependency 25% + Behavioral 35% = 100%.

### 1.3 Why This Architecture

**Windsurf-converted code is predictable.** The conversion was automated and systematic, not a manual rewrite. This means deterministic techniques (name matching, structural fingerprinting, Windsurf metadata extraction) catch 85-90% of mappings with high confidence. Embeddings only add value for the remaining 10-15% where naming or structure diverged significantly.

**Postgres is sufficient for graph queries.** Sentinel’s graph operations are bounded (2-5 hop traversals) and join-heavy (correlating with traceability, gaps, and behavioral tables). Recursive CTEs handle this natively, in the same database, without a separate graph engine to install, backup, or maintain.

**Optionality reduces risk.** Start without embeddings. Run Phases 1-5. Measure gap count. If 50+ items remain unmapped after deterministic matching, enable the embedding flag and run Phase 6. Otherwise, skip it entirely and go to SME review.

### 1.4 Key Technology Decisions

|Decision             |Choice                       |Rationale                                                           |
|---------------------|-----------------------------|--------------------------------------------------------------------|
|COBOL parser         |ProLeap (ANTLR4)             |Native Tandem format, ASG with data/control flow, COPY preprocessing|
|Java indexer         |scip-java                    |Compiler-grade symbol resolution, cross-file references             |
|Dependency graph     |Postgres recursive CTEs      |Zero new infrastructure, joins with existing tables                 |
|Embeddings (optional)|snowflake-arctic-embed-l-v2.0|Text embedder for NL summaries in same vector space both sides      |
|Behavioral testing   |Golden file parity           |Cannot run GnuCOBOL for NonStop dialect                             |

### 1.5 Success Criteria

|Metric              |Target (without embeddings)|Target (with embeddings)|
|--------------------|---------------------------|------------------------|
|Structural coverage |100% mapped                |100% mapped             |
|Dependency integrity|0 broken chains            |0 broken chains         |
|Behavioral parity   |100% tests pass            |100% tests pass         |
|Semantic similarity |N/A                        |Average ≥ 0.85          |
|Composite score     |≥ 95% each layer           |≥ 95% each layer        |
|Audit readiness     |Every mapping traceable    |Every mapping traceable |

## 3. Technology Decisions — Final

### 3.1 COBOL Parsing — ProLeap (unchanged from v2.0)

ANTLR4-based parser with native Tandem format support, AST + ASG with data/control flow, COPY preprocessing, EXEC SQL/ENTER TAL/C extraction as text blocks.

```bash
# Install
git clone https://github.com/uwol/proleap-cobol-parser.git ~/.local/sentinel/proleap
cd ~/.local/sentinel/proleap && mvn clean install -DskipTests
```

### 3.2 Java Parsing — scip-java + Tree-sitter (unchanged from v2.0)

scip-java for compiler-grade symbol resolution, Tree-sitter Java for AST fingerprinting.

### 3.3 Dependency Graph — Postgres Recursive CTEs (NEW)

**Replaces:** Apache AGE

**Why:**

- Zero new infrastructure (already have Postgres)
- Native JOIN with traceability_map, coverage_gaps, behavioral_results tables
- Single backup strategy, single source of truth
- Recursive CTEs handle 2-5 hop traversals efficiently
- No separate Cypher parser to debug

**Data model:**

```sql
CREATE TABLE graph_edges (
    id              SERIAL PRIMARY KEY,
    from_type       TEXT NOT NULL,     -- 'cobol_paragraph', 'java_method', etc.
    from_id         INT NOT NULL,
    to_type         TEXT NOT NULL,
    to_id           INT NOT NULL,
    edge_type       TEXT NOT NULL,     -- 'PERFORMS', 'CALLS', 'CONVERTS_TO', etc.
    metadata        JSONB DEFAULT '{}',
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE (from_type, from_id, to_type, to_id, edge_type)
);
```

**Query patterns covered:**

- Direct edge existence (1-hop lookup) — simple WHERE
- Reachability (n-hop) — recursive CTE
- Chain isomorphism — recursive CTE + EXISTS
- Fan-out/fan-in counts — aggregate with COUNT
- Cross-language correlation — JOIN with CONVERTS_TO edges

### 3.4 Embeddings — Optional (feature-flagged)

**Default: DISABLED**

Controlled by environment variable:

```bash
SENTINEL_ENABLE_EMBEDDINGS=false  # default
```

When disabled:

- `cobol_embeddings` and `java_embeddings` tables are not created
- pgvector extension is not required
- Phase 6 is skipped in the pipeline
- Composite score uses 3-layer formula

When enabled (opt-in):

- Tables created on first run
- pgvector extension verified
- Phase 6 runs after Phase 3
- Composite score uses 4-layer formula

**Model (when enabled):** snowflake-arctic-embed-l-v2.0 (1024 dims) for both sides, operating on natural language summaries.

### 3.5 Behavioral Testing — Golden File Parity (unchanged from v2.0)

Capture I/O from live NonStop system, compare Java output byte-for-byte with COMP-3 epsilon tolerance.

```sql
-- sql/004_graph_edges.sql
SET search_path TO sentinel, public;

-- Central edge table replacing Apache AGE
CREATE TABLE graph_edges (
    id              SERIAL PRIMARY KEY,
    from_type       TEXT NOT NULL
        CHECK (from_type IN (
            'cobol_program', 'cobol_section', 'cobol_paragraph',
            'cobol_copybook', 'cobol_file', 'cobol_data_item',
            'tal_routine', 'c_routine', 'pathway_server',
            'java_class', 'java_method', 'java_field', 'java_interface'
        )),
    from_id         INT NOT NULL,
    to_type         TEXT NOT NULL
        CHECK (to_type IN (
            'cobol_program', 'cobol_section', 'cobol_paragraph',
            'cobol_copybook', 'cobol_file', 'cobol_data_item',
            'tal_routine', 'c_routine', 'pathway_server',
            'java_class', 'java_method', 'java_field', 'java_interface'
        )),
    to_id           INT NOT NULL,
    edge_type       TEXT NOT NULL
        CHECK (edge_type IN (
            -- COBOL edges
            'PERFORMS', 'CALLS', 'COPIES', 'READS', 'WRITES',
            'USES', 'MODIFIES', 'CONTAINS',
            'ENTER_TAL', 'ENTER_C', 'RECEIVES_FROM',
            -- Java edges
            'JAVA_CALLS', 'JAVA_IMPORTS', 'JAVA_EXTENDS',
            'JAVA_IMPLEMENTS', 'JAVA_USES_FIELD', 'JAVA_THROWS',
            'JAVA_CONTAINS',
            -- Cross-language edges
            'CONVERTS_TO', 'REPLACED_BY', 'MIGRATED_TO'
        )),
    metadata        JSONB DEFAULT '{}',
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE (from_type, from_id, to_type, to_id, edge_type)
);

-- Critical indexes for recursive CTE performance
CREATE INDEX idx_ge_from ON graph_edges(from_type, from_id, edge_type);
CREATE INDEX idx_ge_to ON graph_edges(to_type, to_id, edge_type);
CREATE INDEX idx_ge_type ON graph_edges(edge_type);
CREATE INDEX idx_ge_converts ON graph_edges(from_id, to_id) WHERE edge_type = 'CONVERTS_TO';

-- Validation results table
CREATE TABLE dependency_validation_results (
    id                  SERIAL PRIMARY KEY,
    validation_type     TEXT NOT NULL
        CHECK (validation_type IN (
            'call_chain', 'fan_out', 'fan_in', 'data_flow',
            'copybook_dependency', 'inter_program_call',
            'sql_equivalence', 'file_io_pattern',
            'enter_tal_mapping', 'enter_c_mapping',
            'tmf_transaction', 'pathway_ipc',
            'receive_handler', 'guardian_file_access'
        )),
    cobol_source        TEXT NOT NULL,
    java_target         TEXT,
    status              TEXT NOT NULL
        CHECK (status IN ('pass', 'fail', 'warning', 'skipped')),
    detail              JSONB DEFAULT '{}',
    run_id              TEXT NOT NULL,
    run_at              TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_dvr_status ON dependency_validation_results(status);
CREATE INDEX idx_dvr_run ON dependency_validation_results(run_id);
```

### 9.2 Edge Builder

Populates `graph_edges` from ProLeap ASG (COBOL side) and SCIP references (Java side).

```typescript
// src/dependency/edge-builder.ts
export class EdgeBuilder {
    async buildCobolEdges(): Promise<void> {
        // PERFORMS edges from ASG
        const performs = await this.extractPerforms();
        for (const p of performs) {
            await this.insertEdge({
                from_type: 'cobol_paragraph',
                from_id: p.callerId,
                to_type: 'cobol_paragraph',
                to_id: p.calleeId,
                edge_type: 'PERFORMS',
            });
        }

        // ENTER TAL edges
        const talCalls = await this.extractEnterTal();
        for (const t of talCalls) {
            await this.insertEdge({
                from_type: 'cobol_paragraph',
                from_id: t.paragraphId,
                to_type: 'tal_routine',
                to_id: t.talRoutineId,
                edge_type: 'ENTER_TAL',
                metadata: { params: t.params },
            });
        }

        // File I/O, data access, copybook, etc.
        // ... similar patterns
    }

    async buildJavaEdges(): Promise<void> {
        // CALLS edges from SCIP
        const refs = await db.query(
            `SELECT * FROM scip_references WHERE reference_kind = 'reference'`
        );
        for (const r of refs.rows) {
            await this.insertEdge({
                from_type: 'java_method',
                from_id: await this.resolveSymbolToId(r.from_symbol),
                to_type: 'java_method',
                to_id: await this.resolveSymbolToId(r.to_symbol),
                edge_type: 'JAVA_CALLS',
            });
        }
    }

    async buildCrossLanguageEdges(): Promise<void> {
        // CONVERTS_TO from traceability_map
        const maps = await db.query(
            `SELECT * FROM traceability_map WHERE confidence IN ('high', 'medium')`
        );
        for (const m of maps.rows) {
            await this.insertEdge({
                from_type: 'cobol_paragraph',
                from_id: m.cobol_construct_id,
                to_type: 'java_method',
                to_id: m.java_construct_id,
                edge_type: 'CONVERTS_TO',
                metadata: {
                    confidence: m.confidence,
                    mapping_type: m.mapping_type,
                    similarity: m.similarity_score,
                },
            });
        }
    }
}
```

### 9.3 Validation Queries — Recursive CTE Patterns

**Pattern 1: Find broken call chains**

For every COBOL PERFORM edge A→B, verify that the converted Java methods A’→B’ have a corresponding call chain.

```sql
-- src/dependency/chain-validator.ts — broken call chains
WITH broken_chains AS (
    SELECT
        c_edge.from_id AS cobol_caller_id,
        c_edge.to_id AS cobol_callee_id,
        cc_from.construct_name AS caller_name,
        cc_to.construct_name AS callee_name,
        cc_from.program_id
    FROM graph_edges c_edge
    JOIN cobol_constructs cc_from ON cc_from.id = c_edge.from_id
    JOIN cobol_constructs cc_to ON cc_to.id = c_edge.to_id
    WHERE c_edge.edge_type = 'PERFORMS'
    -- Both COBOL paragraphs must be mapped
    AND EXISTS (
        SELECT 1 FROM graph_edges ct1
        WHERE ct1.from_id = c_edge.from_id
        AND ct1.edge_type = 'CONVERTS_TO'
    )
    AND EXISTS (
        SELECT 1 FROM graph_edges ct2
        WHERE ct2.from_id = c_edge.to_id
        AND ct2.edge_type = 'CONVERTS_TO'
    )
    -- But the Java CALL edge between the mapped methods doesn't exist
    AND NOT EXISTS (
        SELECT 1
        FROM graph_edges ct1
        JOIN graph_edges ct2 ON ct2.from_id = c_edge.to_id
            AND ct2.edge_type = 'CONVERTS_TO'
        JOIN graph_edges j_edge ON j_edge.from_id = ct1.to_id
            AND j_edge.to_id = ct2.to_id
            AND j_edge.edge_type = 'JAVA_CALLS'
        WHERE ct1.from_id = c_edge.from_id
        AND ct1.edge_type = 'CONVERTS_TO'
    )
)
SELECT * FROM broken_chains;
```

**Pattern 2: Multi-hop call chain existence (recursive)**

Find all reachable paragraphs from a starting paragraph up to depth 5.

```sql
-- Recursive: all paragraphs reachable from paragraph X
WITH RECURSIVE reachable AS (
    -- Base case
    SELECT from_id AS start_id, to_id AS current_id,
           ARRAY[from_id, to_id] AS path, 1 AS depth
    FROM graph_edges
    WHERE edge_type = 'PERFORMS'
    AND from_id = $1  -- starting paragraph

    UNION ALL

    -- Recursive case
    SELECT r.start_id, e.to_id,
           r.path || e.to_id, r.depth + 1
    FROM reachable r
    JOIN graph_edges e ON e.from_id = r.current_id
        AND e.edge_type = 'PERFORMS'
    WHERE r.depth < 5
    AND NOT (e.to_id = ANY(r.path))  -- cycle prevention
)
SELECT DISTINCT current_id, depth FROM reachable ORDER BY depth;
```

**Pattern 3: Fan-out divergence**

Compare PERFORM fan-out in COBOL vs CALLS fan-out in corresponding Java method.

```sql
WITH cobol_fanout AS (
    SELECT
        cc.id AS cobol_id,
        cc.construct_name,
        cc.program_id,
        COUNT(*) AS cobol_fan_out
    FROM cobol_constructs cc
    JOIN graph_edges ge ON ge.from_id = cc.id
        AND ge.from_type = 'cobol_paragraph'
        AND ge.edge_type = 'PERFORMS'
    GROUP BY cc.id, cc.construct_name, cc.program_id
),
java_fanout AS (
    SELECT
        ct.from_id AS cobol_id,
        COUNT(*) AS java_fan_out
    FROM graph_edges ct
    JOIN graph_edges je ON je.from_id = ct.to_id
        AND je.from_type = 'java_method'
        AND je.edge_type = 'JAVA_CALLS'
    WHERE ct.edge_type = 'CONVERTS_TO'
    GROUP BY ct.from_id
)
SELECT
    cf.construct_name,
    cf.program_id,
    cf.cobol_fan_out,
    COALESCE(jf.java_fan_out, 0) AS java_fan_out,
    cf.cobol_fan_out - COALESCE(jf.java_fan_out, 0) AS delta
FROM cobol_fanout cf
LEFT JOIN java_fanout jf ON jf.cobol_id = cf.cobol_id
WHERE ABS(cf.cobol_fan_out - COALESCE(jf.java_fan_out, 0)) > 2;
```

**Pattern 4: Unmapped ENTER TAL/C calls**

```sql
SELECT
    cc.construct_name,
    cc.program_id,
    cc.nonstop_metadata->>'talRoutine' AS tal_routine,
    cc.nonstop_metadata->>'talSourceFile' AS tal_source
FROM cobol_constructs cc
WHERE cc.construct_type IN ('enter_tal', 'enter_c')
AND NOT EXISTS (
    SELECT 1 FROM graph_edges ge
    WHERE ge.from_type IN ('tal_routine', 'c_routine')
    AND ge.from_id = cc.id
    AND ge.edge_type = 'REPLACED_BY'
);
```

**Pattern 5: Data flow preservation**

For each COBOL paragraph that reads a file, verify the mapped Java method has equivalent I/O.

```sql
WITH cobol_file_reads AS (
    SELECT
        ge.from_id AS paragraph_id,
        cc_p.construct_name AS paragraph_name,
        cc_p.program_id,
        cc_f.construct_name AS file_name,
        cc_f.nonstop_metadata->>'guardianFileName' AS guardian_file
    FROM graph_edges ge
    JOIN cobol_constructs cc_p ON cc_p.id = ge.from_id
    JOIN cobol_constructs cc_f ON cc_f.id = ge.to_id
    WHERE ge.edge_type = 'READS'
)
SELECT
    cfr.paragraph_name,
    cfr.program_id,
    cfr.file_name,
    cfr.guardian_file,
    'FILE_READ_NOT_PRESERVED' AS issue
FROM cobol_file_reads cfr
WHERE EXISTS (
    -- Paragraph is mapped
    SELECT 1 FROM graph_edges ct
    WHERE ct.from_id = cfr.paragraph_id
    AND ct.edge_type = 'CONVERTS_TO'
)
AND NOT EXISTS (
    -- But mapped Java method has no file I/O equivalent
    SELECT 1 FROM graph_edges ct
    JOIN java_constructs jc ON jc.id = ct.to_id
    WHERE ct.from_id = cfr.paragraph_id
    AND ct.edge_type = 'CONVERTS_TO'
    AND (jc.metadata->>'has_file_io' = 'true'
         OR jc.metadata->>'has_jdbc' = 'true')
);
```
